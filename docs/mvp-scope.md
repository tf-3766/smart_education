# MVP Scope

> 第 0 阶段冻结版本。本文定义 18 天实训 MVP 的必须完成、可以简化和延后范围，避免两个人在首版里同时扩张业务和 AI。

## 1. 目标演示链路

MVP 必须能演示下面的闭环：

```text
管理员审核课程
-> 教师创建课程、章节、课时和作业
-> 学生选课并学习章节
-> 学生进入课程论坛发帖或回复
-> 学生使用课程 AI 答疑
-> 学生提交作业
-> 教师评分并使用 AI 生成评语草稿
-> 学生查看成绩统计和学习风险
-> 学生查看考试安排
-> 教师查看智能组卷建议并保存试卷草稿
```

## 2. 技术范围

| 领域 | MVP 决策 |
|---|---|
| 架构 | `edu-gateway` + `edu-biz-service` + `edu-ai-service`，不再拆更多微服务 |
| 后端底座 | JDK 21、Spring Boot 3.x |
| 微服务组件 | Spring Cloud Alibaba 只使用 Nacos、Gateway、OpenFeign |
| 数据库 | MyBatis-Plus + MySQL 8.0 + Flyway |
| 缓存和消息 | Redis + RabbitMQ |
| AI | 首版 Fake Adapter；真实实现默认 Spring AI + Qdrant |
| CI/CD | 当前继续 GitHub 远端，只补 `.gitlab-ci.yml` 作为 GitLab CI/CD 交付物 |

## 3. 必须完成

| 功能 | Owner | MVP 标准 |
|---|---|---|
| 课程审核 | A | 管理员可审核，教师可看到审核结果 |
| 课程、章节、课时 | A | 教师可维护课程内容，学生可读取已发布内容 |
| 学生选课和学习进度 | A | 学生可选课、开始/完成课时，系统可计算课程进度 |
| 作业发布 | A | 教师可创建并发布作业，包含截止时间和基础附件元数据 |
| 作业提交 | A | 学生可草稿保存、正式提交，系统校验截止时间和课程权限 |
| 评分和成绩发布 | A | 教师可评分、保存评语、发布成绩，学生只看已发布成绩 |
| 成绩统计 | A | 教师可查看课程作业成绩列表、平均分、未交/低分统计 |
| 学习预警 | A | 系统基于进度、缺交、低分生成正式预警记录 |
| 课程论坛 | A | 列表、发帖、回复、教师或管理员删除 |
| 课程 AI 答疑 | B | Fake Adapter 返回回答、引用或无来源提示；权限由 Biz context 控制 |
| 章节知识点摘要 | B | AI 返回摘要草稿，教师确认后由 Biz 保存或发布 |
| AI 批改评语草稿 | B | AI 返回评语草稿，教师确认后由 Biz 写入正式评语 |
| 考试安排 | B | 教师可创建考试安排，学生可查看 |
| 题库和智能组卷建议 | B | 教师维护最小题库，AI 返回组卷建议，教师保存试卷草稿 |
| Gateway 和联调 | B | Biz/AI 路由、JWT、CORS、AI 限流、SSE 转发可验证 |
| Docker、演示数据、启动文档 | A+B | B 主负责 Docker/启动文档，A 主负责 Biz seed |

## 4. 可以简化

- AI 使用 Fake Adapter，不要求首版接真实模型 key。
- RAG 可以先使用固定引用或 fake vector result，接口结构必须和真实 Qdrant 实现兼容。
- 文件上传先保存附件元数据和 URL，不做完整对象存储、病毒扫描、签名 URL。
- 学习预警先用规则：进度落后、缺交、低分。
- 考试只做到安排、题库、试卷草稿，不做在线答题会话。
- 课程论坛只做基础讨论和删除，不做举报、敏感词、推荐流和复杂审核。
- 管理员统计只做支撑演示的聚合，不做大屏。
- RabbitMQ 可先用于异步任务边界设计，首版关键闭环允许同步调用。

## 5. 延后开发

- 完整在线考试引擎、答题保存、自动交卷和阅卷。
- 考试成绩进入课程总成绩。
- 自动批量批改。
- AI 管理后台。
- 复杂学习风险模型。
- 大规模向量索引管理后台。
- 全功能消息中心。
- 复杂论坛治理。
- 真实对象存储和文件安全扫描。
- 多租户、组织架构、班级复杂权限。
- Sentinel、Seata 等非要求组件。
- 为了“看起来微服务化”继续拆服务。

## 6. 成绩和预警边界

| 概念 | Owner | 数据边界 |
|---|---|---|
| 作业成绩 | A | 来源于作业提交和教师评分；保存、发布、更正都在 `assignment/grade` 模块 |
| 考试成绩 | B | 首版不进入课程总成绩；如后续接入，必须通过 A 提供的成绩接口或事件 |
| 课程总成绩 | A | 首版聚合已发布作业成绩；不直接读取 B 的考试内部表 |
| 学习预警 | A | 正式预警记录、证据、状态由 Biz 保存；AI 只生成解释和建议草稿 |
| AI 评语 | B 提供草稿，A 保存事实 | 教师确认前不是正式评语；确认后由 A 写入成绩/评语记录 |
