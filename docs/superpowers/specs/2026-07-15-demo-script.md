# 项目演示脚本（按后端真实流程）

三个账号轮换演示：管理员 → 教师 → 学生 → 教师 → 学生。每步都是真实后端调用。
前置：后端各服务健康（网关 18080）；如演示 AI，需 AI 服务正常 + 已配置密钥。

## 第一幕 · 管理员：平台就绪（约 3 分钟）

1. 管理员登录 → **数据看板**：平台总览（用户/课程/待办）。
2. **用户管理**：审批一名「待审核」教师（注册审批流）→ 状态变已启用。
3. **AI 设置**：填入百炼密钥、选模型（qwen-plus）→「测试连接」→ 状态卡显示提供方/模型/可用。
4. **内容治理**：发布一条全体公告（稍后学生端站内信可见，呼应实时通知）。

## 第二幕 · 教师：备课（约 6 分钟）

5. 教师登录 → **课程管理 → 新建课程**：从**内置课程库**选一门模板（编码/名称/简介自动回填），
   设学期与选课时间窗 → 创建草稿 → **提交审核**。
6. 切管理员 → **课程治理**：待审核列表看到该课 → 填意见 → **通过**。
   （可顺带演示按状态浏览全部课程。）
7. 切教师 → **课程管理**：**发布**课程 → 进入**课程内容**：新建章节 → 新建课时（图文/视频）
   → 发布课时与章节 → 上传/外链一份资料。
8. **作业批改 → 新建作业**：填作业内容与要求、满分、截止时间 → **发布**。
9. **考试题库**：新建题库 → **新建题目**（单选/判断/简答各一）→ 新建考试（时间窗/总分）
   → **组卷**（勾题配分=总分）→ **发布试卷**。

## 第三幕 · 学生：学习闭环（约 6 分钟）

10. 学生登录 → 站内信铃铛：收到系统公告（第 4 步）。
11. **选课中心**：搜到新课程 → **加入学习** → **我的课程**：进入学习。
12. 课时页：阅读内容 → **完成课时**（进度上报）→ 右侧**课程答疑（AI）**提问，流式回答+引用。
13. **学习任务**：找到作业 → 填写答案 → **确认提交**。
14. **考试安排**：进入考试 → 答题 → **交卷**（客观题自动判分）。
15. **课程互动**：发一个讨论主题。

## 第四幕 · 教师：批改与干预（约 5 分钟）

16. 教师 → **作业批改**：打开学生提交 → **AI 评语草稿** → 采用到评语 → 打分 → **评分并发布**。
17. **考试题库 → 答卷**：进入阅卷 → 为简答题人工评分 → 提交（总分回填）。
18. **学情预警**：选课程 → **生成预警**（进度<50%/缺交/低分三规则）→ 对一条预警
    **生成 AI 解读** → 采用为干预记录 → **记录干预并关闭**。
19. **课程互动**：回复学生主题（教师身份标识）。

## 第五幕 · 学生收尾（约 2 分钟）

20. 学生 → 站内信：收到成绩发布/公告通知 → 点击跳转对应页面。
21. **成绩进度**：看到作业得分、评语与考试成绩；首页进度更新。

## 讲解要点（可穿插）

- 课程生命周期：草稿 → 审核 → 发布 → 下线（管理员总览可见全链路）。
- 权限边界：学生不可见未发布内容；下线课程学生不可再进入。
- AI 定位：只产草稿/建议，正式数据（评语/干预）须教师确认后写入。
- 预警规则：进度（<25% 高 / <50% 中）、缺交（中）、低分（<40% 高 / <60% 中）。

## 应急预案

- AI 服务不可用 → 跳过 12/16/18 的 AI 环节，其余流程不受影响（按钮有明确错误提示）。
- 时间紧 → 最小闭环：5 → 6 → 7 → 8 → 11 → 13 → 16 → 21（建课到成绩，约 10 分钟）。
