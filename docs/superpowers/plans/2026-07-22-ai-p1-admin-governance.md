# AI P1 管理端治理增强进度

日期：2026-07-22
状态：本切片已实现并验证

## 已完成

- 新增统一治理预审草稿接口，管理员一次可提交教师注册和课程共 1-50 项。
- 教师注册预审只核对当前待审核快照并输出逐项风险、建议和人工复核原因；不提供批量通过或批量驳回。
- 课程合规检查基于授权业务上下文核对审核状态、课时和资料完整度，输出准备度、问题码和整改建议。
- 用户管理页接入“教师注册批量预审”，课程治理页接入“课程合规检查”，原逐项审批入口保持不变。
- 能力目录新增 `admin.teacher-registration.batch-precheck` 与 `admin.course.compliance-check`；管理员自然语言助手可先查询真实 ID，再调用 `draftAdminGovernanceReview` 生成与页面相同协议的结构化结果卡。
- 配置模型时，模型基于授权证据和管理员标准补充风险码与原因；确定性规则始终保留且不可被模型删除。未配置模型时明确返回 `FRAMEWORK_ONLY`。
- 课程检查覆盖课时/资料数量、课时正文、资料抽取正文、审核状态和生命周期，并返回成功数、失败数、逐项证据与失败原因。
- Biz 到 Feign 再到 AI 的待审核教师快照已升级为类型化 DTO，携带账号、姓名、注册时间和目标版本，不再解析展示字符串。
- 课程合规上下文及公开预审结果已补充简介、分类、学期、院系、学分、课程时间窗、选课时间窗和目标版本；缺失任一时间端点或时间顺序非法均进入确定性问题码与扣分。
- 前端真实接口类型、演示数据、演示规则和管理页面逐项对齐后端字段；教师与课程预审结果均展示目标版本，教师结果同时展示注册时间。
- 修复批量辅助批改面板两个表单标签缺少 `for/id` 的可访问性回归。

## 安全边界

- 本切片只产生草稿，不写教师审核结果，不发布或下线课程。
- 正式教师审批和课程审批继续走 Biz 原接口、角色校验和人工逐项确认。
- 不在上下文中的教师返回 `NOT_ELIGIBLE`；课程事实通过管理员授权的 Biz 上下文读取，不接受前端拼装事实。
- 未配置外部模型时返回 `FRAMEWORK_ONLY`，仍执行确定性的规则检查；不得把规则兜底描述为模型已运行。
