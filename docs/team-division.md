# 三人分工与协作基线

> 团队：后端 A、后端 B、前端 C
> 当前阶段：先完成已冻结的课程主链和作业主链；AI/RAG 暂不进入业务代码实现。

## 1. 总体分工

| 成员 | 主要职责 | 主要目录 | 当前优先交付 |
|---|---|---|---|
| 后端 A | Biz 主链、权限、课程、作业、成绩、论坛、预警 | `backend/edu-biz-service` 的 `auth`、`course`、`assignment`、`grade`、`forum`、`warning`、`shared` | 作业发布、学生提交、教师批改、学生成绩查询 |
| 后端 B | 考试题库、AI 服务边界、网关、服务间契约、部署联调 | `edu-ai-service`、`edu-gateway`、`edu-feign-api`、`edu-biz-service/exam`、`deploy` | 考试题库和考试安排；维护 Gateway、Compose 和接口契约 |
| 前端 C | Vue 页面、API Client、路由守卫、联调和演示流程 | `src/**`、`ui-spec.md`、`wireframes.md`、`sitemap.md` | 登录、课程、学习页与课程审核接入真实接口 |

## 2. 后端 A：业务主链

### 已有代码维护

- 认证、JWT、角色权限和课程/章节/课时/资料/选课/学习进度。
- `online_education_bootstrap.sql` 中 Biz 表结构和演示数据的一致性。
- 课程审核流程与管理员接口。

### 新任务顺序

1. 作业：教师创建、修改、发布；学生列表、草稿/提交和截止校验。
2. 成绩：教师查看提交、评分、发布；学生只读已发布成绩。
3. 论坛：课程成员发帖、回复；教师/管理员治理可见性。
4. 预警：基于缺交、低分、进度生成记录；学生查看、教师处理。

### 验收标准

- 每个接口都校验登录角色、课程归属、对象状态和资源范围。
- 所有新表或字段同步更新 Bootstrap SQL、API 契约和测试数据。
- 前端可经 Gateway 完成“教师发布作业 -> 学生提交 -> 教师评分 -> 学生查看成绩”闭环。

## 3. 后端 B：考试、平台与 AI 边界

### 当前优先任务

1. 考试与题库：题库、题目、考试安排、试卷草稿的最小 CRUD。
2. 平台联调：维护 Gateway 路由、JWT 透传、CORS、traceId、Redis 限流和 Compose。
3. 服务间契约：维护 `edu-feign-api`；需要 Biz 数据时只通过 Feign DTO，不直接使用 Biz Mapper。
4. 文档与接口：维护 OpenAPI 和待实现契约，接口完成后将状态改为“已实现”。

### AI 暂缓范围

- 已接入 Spring AI 1.1.8 和阿里云百炼 OpenAI 兼容接口，完成课程问答 SSE、课时摘要、作业评语、风险解释、组卷建议、模型未配置降级和管理状态。
- AI 必须经 `edu-feign-api` 读取 Biz 授权上下文，不连接 Biz MySQL，不直接修改正式数据。
- 当前问答使用 Biz 返回的授权课时正文；Qdrant/向量 RAG 仍作为后续独立能力，不影响现有受控上下文生成。

### 验收标准

- 教师能维护题库、创建考试、保存试卷草稿；学生能查看本人课程的考试安排。
- Gateway 只暴露公开 `/api/v1/**`，内部 `/_internal/**` 不对前端开放。
- Compose、环境变量、启动说明和实际端口保持一致。

## 4. 前端 C：页面与联调

### 当前优先任务

1. 建立 API Client：统一注入 Token、处理 `401/403/409`、读取统一响应和 traceId。
2. 登录与路由：登录页、角色入口、未登录跳转、错误和加载状态。
3. 课程主链：学生课程目录、我的课程、学习大纲、课时学习；教师课程创建和内容维护；管理员课程审核。
4. 作业主链：后端 A 完成接口后接入学生提交、教师批改和成绩查看。

### 开发边界

- 当前只调用 [API 参考](./api-reference.md) 标为“已实现”的接口。
- 待实现模块使用独立 Mock，不在页面中伪装成已联调完成。
- `ui-spec.md`、`wireframes.md`、`sitemap.md` 是设计输入；实际页面和 API 状态以 `src/**` 与 API 参考为准。

### 验收标准

- 三类角色都有可进入的页面、加载态、空态、错误态和无权限提示。
- 前端仅请求 Gateway `http://localhost:18080`，不直连 Biz、AI 或数据库容器。
- 演示时可完成登录、课程审核、选课、课时学习和作业闭环。

## 5. 协作规则

| 事项 | 规则 |
|---|---|
| 分支 | 从最新 `dev` 创建 `feature/*` 或 `fix/*`；功能完成后合回 `dev`，由 `dev` 发版到 `main`。 |
| 接口 | 先改 API 契约/OpenAPI，再实现后端和前端；破坏性字段变更必须三人确认。 |
| 数据库 | 只维护 `online_education_bootstrap.sql`；变更前明确表归属，变更后在空 MySQL 8.4 验证。 |
| 公共模块 | `edu-common`、`edu-feign-api`、Gateway 安全配置、Compose 和 API 文档必须单独说明影响范围。 |
| 联调 | 前端只访问 Gateway；后端 B 访问 Biz 仅经 Feign；不得跨服务引用 Mapper 或 Entity。 |

## 6. 联调节奏

1. A 维护已实现的认证和课程接口，C 先接入这些真实接口。
2. A 完成作业和成绩的一个完整接口组后，C 在同一轮接入；不要等待论坛和预警全部完成。
3. B 并行完成考试题库和平台配置；C 对未完成接口继续使用 Mock。
4. 最后统一执行 Gateway 端到端联调、演示数据校验和答辩流程演练。

本文是当前执行优先级、模块边界和联调规则的唯一基线。
