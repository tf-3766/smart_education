# 三人分工与协作基线

> 团队：后端 A、后端 B、前端 C
> 当前阶段：三类用户、课程/作业/考试/治理主链已可联调；AI 进入 Spring AI 框架和授权上下文阶段。

## 1. 总体分工

| 成员 | 主要职责 | 主要目录 | 当前优先交付 |
|---|---|---|---|
| 后端 A | Biz 主链、权限、课程、文件、作业、成绩、论坛、预警 | `backend/edu-biz-service` 的 `auth`、`storage`、`course`、`assignment`、`grade`、`forum`、`warning`、`shared` | 已完成主链，进入联调与缺陷修复 |
| 后端 B | 考试题库、平台治理、AI 服务边界、网关、服务间契约、部署联调 | `edu-ai-service`、`edu-gateway`、`edu-feign-api`、`edu-biz-service/exam`、`edu-biz-service/platform`、`deploy` | 维护已实现的考试/公告/分类/统计，扩展授权 AI 能力 |
| 前端 C | Vue 页面、API Client、路由守卫、联调和演示流程 | `frontend/**`、`ui-spec.md`、`wireframes.md`、`sitemap.md` | 维护现有 Vue 3 前端并完成真实接口联调 |

## 2. 后端 A：业务主链

### 已有代码维护

- 登录注册、JWT、学生/教师角色、超级管理员授权，以及课程/章节/课时/资料/选课/学习进度。
- `online_education_bootstrap.sql` 中 Biz 表结构和演示数据的一致性。
- 课程审核流程与管理员接口。

### 已完成任务

1. 作业：教师创建、修改、发布；学生列表、草稿/提交和截止校验。
2. 成绩：教师查看提交、评分、发布；学生只读已发布成绩。
3. 论坛：课程成员发帖、回复；教师/管理员治理可见性。
4. 预警：基于缺交、低分、进度生成记录；学生查看、教师处理。
5. 文件与头像：统一文件元数据、本地存储适配、授权下载、头像绑定，以及课程资料和作业文件引用。

当前 Backend A 的上述接口已实现并有集成测试；后续以接口联调、缺陷修复和数据库空库验证为主，不再把这些模块标记为 Mock。

### 验收标准

- 每个接口都校验登录角色、课程归属、对象状态和资源范围。
- 所有新表或字段同步更新 Bootstrap SQL、API 契约和测试数据。
- 前端可经 Gateway 完成“教师发布作业 -> 学生提交 -> 教师评分 -> 学生查看成绩”闭环。

## 3. 后端 B：考试、平台与 AI 边界

### 已完成的非 AI 任务

1. 题库、题目、考试安排、试卷编排与发布。
2. 学生答题、时间窗、客观题判分和简答题教师批阅。
3. 课程/系统公告、课程分类治理和管理员聚合统计。
4. Gateway 路由、JWT 透传、CORS、traceId、AI 限流和 Compose；AI 访问 Biz 时透传当前用户 JWT。

### AI 当前边界

- 已接入 Spring AI 1.1.8 和阿里云百炼 OpenAI 兼容接口，完成课程问答 SSE、课时摘要、作业评语、风险解释、组卷建议、模型未配置降级和管理状态。
- AI 必须经 `edu-feign-api` 读取 Biz 授权上下文，不连接 Biz MySQL，不直接修改正式数据。
- 当前问答使用 Biz 返回的授权课时正文；Qdrant/向量 RAG 仍作为后续独立能力，不影响现有受控上下文生成。

### 验收标准

- 教师能维护题库、发布试卷并批阅简答题；学生能查看安排、开始考试、提交答卷和查看已评结果。
- 教师/管理员能发布并撤回公告，管理员能管理课程分类并查看平台统计。
- AI 课程问答和课时摘要继承当前用户的课程资源范围，未配置模型时不伪造结果。
- Gateway 只暴露公开 `/api/v1/**`，内部 `/_internal/**` 不对前端开放。
- Compose、环境变量、启动说明和实际端口保持一致。

## 4. 前端 C：页面与联调

### 当前优先任务

1. 维护 API Client：统一注入 Token、处理 `401/403/409`、读取统一响应和 traceId。
2. 认证与路由：维护登录、个人中心、角色入口、未登录跳转、错误和加载状态。
3. 课程主链：学生课程目录、我的课程、学习大纲、课时学习；教师课程创建和内容维护；管理员课程审核。
4. 作业、考试与治理主链：接入学生提交、教师批改、考试答题、公告和管理统计。

### 开发边界

- 当前只调用 [API 参考](./api-reference.md) 标为“已实现”的接口。
- 待实现模块使用独立 Mock，不在页面中伪装成已联调完成。
- `ui-spec.md`、`wireframes.md`、`sitemap.md` 是设计输入；实际页面和 API 状态以 `frontend/**` 与 API 参考为准。

### 验收标准

- 三类角色都有可进入的页面、加载态、空态、错误态和无权限提示。
- 前端仅请求 Gateway `http://localhost:18080`，不直连 Biz、AI 或数据库容器。
- 演示时可完成登录、课程审核、选课、课时学习和作业闭环。

## 5. 协作规则

| 事项 | 规则 |
|---|---|
| 分支 | 从最新 `dev` 创建 `feature/*` 或 `fix/*`；功能完成后合回 `dev`，由 `dev` 发版到 `main`。 |
| 接口 | 先改 API 契约/OpenAPI，再实现后端和前端；破坏性字段变更必须三人确认。 |
| 数据库 | 只维护 `online_education_bootstrap.sql`；变更前明确表归属，变更后在空 MySQL 8.4 验证。 |
| 公共模块 | `edu-common`、`edu-feign-api`、Gateway 安全配置、Compose 和 API 文档必须单独说明影响范围。 |
| 联调 | 前端只访问 Gateway；后端 B 访问 Biz 仅经 Feign；不得跨服务引用 Mapper 或 Entity。 |

## 6. 联调节奏

1. A 维护认证、课程、作业、成绩、论坛和预警闭环，并优先处理联调缺陷。
2. B 维护考试、公告、课程分类、统计和 AI 服务边界，继续迭代 RAG 等后续能力。
3. C 按 [API 参考](./api-reference.md) 的“已实现”标记接入真实接口，仅对明确待实现的 AI 扩展使用 Mock。
4. 三方统一执行 Gateway 端到端联调、演示数据校验和答辩流程演练。

本文是当前执行优先级、模块边界和联调规则的唯一基线。
