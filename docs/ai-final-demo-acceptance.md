# AI 最终演示验收手册

本手册面向最终演示：学生只问答；教师和管理员可以问答、生成安全草稿，并发起必须确认的正式动作。前端入口、AI 能力注册、模型工具和 Biz 权限使用同一角色边界。

## 1. 演示前准备

### 1.1 账号

| 角色 | 用户名 | 密码 | 预期权限 |
|---|---|---|---|
| 学生 | `student` | `123456` | 仅本人数据与授权课程问答，无草稿/动作按钮 |
| 教师 | `teacher` | `t123456` | 负责课程问答、教学草稿、待确认教学动作 |

| 普通管理员 | `admin_ops` | `admin123` | 平台与课程治理；不含用户明细、教师预审和教师审批 |

### 1.2 数据库升级

已有 MySQL 数据卷应先备份并由 `online_education_bootstrap.sql` 重建；如需额外的综合展示记录，再执行 `2026-07-19-demo-showcase.sql`。

完整 UTF-8 导入命令见 [本地开发说明](./local-development.md#5-sql-初始化)。导入后应存在待审核教师 `teacher_pending`、课程 `21001`、3 份未发布成绩的提交和开放预警。

## 2. 当前 AI 功能总表

### 2.1 学生：只读问答

- 查询本人课程、选课窗口、章节、已解锁课时、学习进度。
- 查询本人待办作业、考试安排、已发布成绩、教师反馈和学习预警。
- 查询本人可见公告、讨论和通知。
- 在授权课程内基于课时正文和资料正文进行 RAG 流式问答，并展示知识库引用。
- 不注册任何 `DRAFT`/`ACTION` 能力，不提供确认、取消、重试或写业务工具。

### 2.2 教师：问答、草稿与受控自动流

- 查询负责课程、章节、课时、资料、作业、考试、提交、成绩、预警、公告和教学概况。
- 课程 RAG 问答：按课程归属读取课时/资料正文，流式返回并展示引用。
- 生成课时摘要、单份提交评语与成绩建议、组卷结构建议。
- 最多 50 份批量辅助批改：结构化建议分数、评语、置信度；空答案、过短、越界、模型异常和低置信度自动进入复核队列；已发布成绩不参与；不提供批量保存/发布。
- 生成学生预警解读和“提醒 → 补救材料 → 补交任务 → 复查”干预计划。
- 一句话按课程资料与章节创建 AI 题库草稿及题目；兼容模型常见题型、选项和答案 JSON 结构。
- 生成作业草稿、考试框架草稿、公告草稿。
- 一键生成“资料梳理 → 教案 → 摘要 → 作业 → 题目 → 公告”的教学包执行计划。
- 对已有草稿作业发起发布动作；对单份提交发起保存或发布成绩动作。正式写入必须逐项预览和确认。
- AI 结果携带业务类型、资源 ID 和深链，可跳转题库、考试、公告或批改页并定位目标。

### 2.3 管理员与超级管理员：治理问答与受控执行

- 查询平台用户/课程/作业/考试/预警运营指标、内容治理状态和 AI 服务状态。
- 生成每日运营简报、异常信号和需要人工确认的处置建议。
- 最多 50 项教师注册批量预审与课程合规检查；返回证据、风险码、准备度和复核建议，不自动审批或发布课程。
- 创建或调整学期统一选课窗口，进入高风险强确认动作。
- 超级管理员可审批或驳回教师注册；普通管理员不接收用户明细、教师预审或审批能力。
- 选课窗口和教师审批必须输入“确认执行”，并重新校验目标版本。

### 2.4 通用执行与安全能力

- SSE 流式回答，展示 `meta/capability/tool/delta/citation/action/done` 过程事件。
- JWT 角色、本人数据和课程归属三层权限隔离；AI 服务不直接连接 Biz 数据库。
- 草稿统一标记 `source=AI`，默认不发布；正式动作统一进入待确认队列。
- 完整七阶段：计划 → 风险 → 预览 → 确认 → 执行 → 验证 → 失败补偿。
- 支持确认、取消、过期、失败重试、幂等键、目标版本冲突检查、执行结果深链。
- 日期时间与天气问答；全局助手会根据问题只读取必要事实域，未读取的数据不会误报为“没有”。
- 教师/管理员的“完整概况、平台汇总、待审核教师”等全局只读问题不受当前页面限制；模型暂时失败时仍返回已授权的实时数据库事实，写操作不降级执行。
- 管理员运营简报严格区分“实时聚合指标”“未提供的明细”和“经规则确认的异常”：未提供明细不得解释为底层表为空或指标失真；同一课程定义跨教师/学期开课、DRAFT 未提交审核、OFFLINE 计入课程总数均按正常业务口径解释。
- 纯文字建议不属于待确认动作；只有工具真实返回 `actionId`、风险、预览和确认策略后才能显示 `WAITING_CONFIRMATION`。未接入执行器的预警重算、课程归档/下线、批量通知和公告受众修改不得生成伪确认口令。

## 3. 自动化验收

先确认三个后端服务均已启动，再在项目根目录运行：

```powershell
python -m unittest tests.integration.test_ai_smoke
python tests/integration/ai_smoke.py
python tests/integration/ai_acceptance.py
```

预期：

- 单元测试全部通过。
- `ai_smoke.py` 基础模型链路为 `9 PASS · 0 FAIL · 0 SKIP`（演示数据完整时）。
- `ai_acceptance.py` 覆盖学生/教师/普通管理员/超级管理员能力矩阵、四组越权拒绝、三类全局问答、教师一句话生成 3 道题及深链、教学包、风险干预、批量批改、运营简报、治理预审，以及高风险计划的错误口令拒绝与取消；完整演示数据下应为 `22 PASS · 0 FAIL · 0 SKIP`。
- 验收脚本会创建一个带进程号的 AI 草稿题库和一个随后取消的高风险计划；不会发布资源、审批教师或确认任何正式业务动作。

## 4. 明天浏览器手工测试顺序

### 4.0 课程定义、真实资料与知识库回归

1. 使用 `student_d / 123456` 打开 `/student/lessons/22404`；预期课程为“数据结构与算法”，正文区直接显示“数据结构思维导图”，不再出现 HTTP 404。
2. 打开右下角全局助手并输入：`根据当前资料解释顺序表和链表的区别。`；预期助手自动带入课程 `22002` 和课时 `22404`，回答引用“顺序表与链表”和“数据结构思维导图”。
3. 使用 `teacher_cs / t123456` 进入数据结构课程内容页，点击“同步知识库”；预期成功且至少索引 1 个片段，随后课程问答可以命中资料正文。
4. 使用 `teacher_math / t123456` 进入课程管理；应能看到 `DEMO-JAVA-001` 的独立 `2027-SPRING` 草稿开课。再次选择任一已审核课程定义时，编码、名称和简介只读，但学期、学分等开课字段可独立填写，不出现“课程编号已存在”。
5. 如需验证新课程沉淀，创建唯一测试编码、提交管理员审核并通过；随后其他教师的新建课程下拉框应出现该定义。审核前不得出现。

### 4.1 学生

1. 使用 `student` 登录，打开右下角全局 AI 助手。
2. 确认能力卡全部是只读问答，页面中没有“生成草稿”“确认执行”“重试动作”等入口。
3. 输入：`请汇总我的课程、待办作业、考试、已发布成绩和学习预警，只查询不修改数据。`
4. 进入一门已发布课程或课时，输入：`请根据本课时和课程资料解释核心概念，并列出引用来源。`
5. 预期：逐字流式显示；可展开工具调用；课程问答显示引用；不得出现其他学生数据或未发布成绩。

### 4.2 教师问答与草稿

1. 使用 `teacher_math / t123456` 进入课程管理，编辑 `2026 秋季` 的 Java 草稿；预期学分保持 `2`，继续显示“复用管理员统一选课时间”和 `2026/07/01 09:00 ~ 2026/08/05 23:59`，不会变成 3.5 学分或自定义时间。未提交/已驳回草稿显示“删除草稿”，其他状态不显示。
2. 分别进入作业、考试、学情预警和课程互动；课程下拉框只能出现审核通过且状态为 `PUBLISHED/ONGOING/FINISHED` 的正式课程，Java 草稿不得出现。
3. 使用 `teacher` 登录，输入：`请汇总我的完整教学概况、待批改提交和学生预警，只查询不修改数据。`
4. 预期：显示真实课程、作业、考试、待批改提交和预警数量，不得把未请求事实域误报为空。
5. 进入课程内容页，点击教学包计划；应生成六阶段计划且不直接创建/发布正式资源。
6. 在全局助手输入：`根据已发布测试课程的章节、课时和资料正文，创建3道单选题草稿题库，题库名为明日验收AI题库。`
7. 点击结果深链；应进入考试题库页并定位 `source=AI`、状态为草稿的题库，包含 3 道题。
8. 打开考试组卷，生成 AI 组卷建议后点击“采用建议”；预期题目自动勾选、分值立即写入，合计等于考试总分；核对后点取消，不保存测试试卷。
9. 进入预警页，分别生成预警解读和干预计划；应包含提醒、补救、补交和复查。采用后干预记录最多 4000 字，点击“记录干预并关闭”不再出现参数错误。
10. 进入作业批改工作台，选择最多 50 份未发布成绩的提交，填写评分标准和 `0.75` 复核阈值后生成建议。
11. 预期：每份有分数、评语、置信度；异常/低置信度进入复核；点击建议回到原始提交；没有批量保存或自动发布入口。

### 4.3 教师待确认动作

1. 对已有草稿作业说：`请发布该作业，先给我预览，不要直接执行。`
2. 对一份待批改提交说：`建议评分并保存成绩，先生成待确认计划。`
3. 预期：出现风险、预览、目标版本和确认按钮；取消后状态为 `CANCELLED`；刷新助手仍可看到动作记录。
4. 演示确认时只选择专用演示数据。执行成功后应显示验证结果和业务页深链；再次确认同一动作不得重复写入。

### 4.4 管理员与超级管理员

1. 先使用 `admin_ops` 登录；确认可见平台/课程/内容/统计/AI 状态与选课窗口能力，但看不到用户治理、教师批量预审和教师审批能力；直接访问用户明细应返回 403。
2. 再使用 `admin` 登录，输入：`汇总平台用户、课程、运营指标、异常信号、待审核教师和 AI 服务状态，只查询不执行。`
   预期用户角色汇总与用户管理页一致（当前演示数据为学生 9、教师 5、管理员用户 2），并明确返回 `teacher_pending`；不得因当前页面或模型自行计数而报告为 0。
3. 输入：`根据刚生成的每日运营简报，先列出异常项的处理计划、风险等级和预览变化。不要直接执行。`；预期开放预警聚合值不会被解释为“指标失真”，跨学期开课与 DRAFT/OFFLINE 状态不会被误报为异常，也不会出现 `ARCHIVED`、`确认执行 A/B/C` 或无 `actionId` 的伪 `WAITING_CONFIRMATION` 计划。
4. 在管理首页生成每日运营简报；应包含指标、经证据确认的异常或“当前未发现”，不直接操作业务。
5. 在用户管理页对 `teacher_pending` 执行 AI 批量预审；在课程审核页执行课程合规检查。
6. 预期：显示证据、风险码、准备度与人工复核建议；教师仍保持 `PENDING`，课程状态不被修改。
7. 输入：`为 teacher_pending 生成通过审核计划，但不要直接执行。`
8. 预期：动作状态 `WAITING_CONFIRMATION`、风险 `HIGH`、策略 `STRONG_CONFIRM`；未输入“确认执行”不能执行。
9. 取消一次验证 `CANCELLED`；如需演示成功执行，再创建新计划、核对版本后输入“确认执行”。
10. 打开选课窗口页创建调整计划，验证同样的强确认、版本冲突和结果跳转。

## 5. 最终通过标准
