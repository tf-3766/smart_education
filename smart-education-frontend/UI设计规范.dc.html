05 · 顶栏与身份切换

  • 顶栏只保留：Logo、全局搜索、预览身份（弱化为文字+下拉，非按钮）、用户头像 —— AI 助手入口只在业务流程内出现，不与顶栏/首页 CTA 重复
  • "预览身份"标注为演示态文案，与真实权限切换在视觉上区分（详见用户与权限页的角色/资源范围矩阵）
  • 高度 56px，描边分隔，不用阴影

05b · 控制台式页面结构

借鉴企业级云控制台的三层导航范式：全局工具栏（不变）→ 业务域切换（仅首页出现）→ 进入具体业务后替换为该业务自己的侧边栏。层与层之间用字重/颜色而非色块、边框做区分，视觉上极度克制。

导航三层状态

层级出现位置规则
① 全局工具栏所有页面，56px，全局唯一Logo + 平台名、全局搜索（居中，最大宽 400px，浅灰底无边框）、预览身份、头像；不随页面变化，不重复展示当前页面标题
② 业务域 Tab仅首页/总览层级，44px横向 2-4 个域名（如"工作台 / 教学业务 / 管理"）；当前域 teal-700 文字+加粗 + 底部 2px teal-600 描边，其余 gray-6 常规字重 —— 不用背景色块，纯靠字重和下划线区分，这是与侧边栏当前态（用背景块）的关键差异
③ 业务侧边栏进入具体业务页面后，替换②，196px顶部不再是平台 Logo，而是"‹ 返回 + 业务域名称"作为侧边栏标题；下方是该业务域内的扁平页面列表（无分组大写标题）；当前页 teal-50 背景 + 左侧 3px teal-600 内阴影

即：首页 = 工具栏 + 域 Tab + 内容（无侧边栏）；业务页 = 工具栏 + 业务侧边栏 + 内容（无域 Tab）。两者不同时出现。

筛选行（列表/表格页必备）

  • 位置：页面标题下方、表格上方，独立一张浅灰底描边卡片，与内容区背景区分
  • 时间类筛选用胶囊分段控件（如"近1天 / 近7天"），当前项 teal-600 描边+白底；其余筛选为下拉框
  • 组成：搜索框（最大宽 220px）+ 1-2 个下拉筛选 + 主操作「查询」（teal-600 实心）+ 次操作「重置」（描边）
  • 适用：成绩统计、考试与题库、用户与权限等表格为主的页面；详情/线程类页面（批改详情、论坛帖子）不需要

信息卡片的"底部链接"样式

  • 入口类卡片（如首页的功能引导卡）：标题 + 两行说明 + 1px 分隔线 + 底部一个 teal-600 文字链接（"新建课程 ›"），卡片本身仅描边、无强调底色
  • 右侧信息栏（仅首页）：用量卡片标题旁挂"本月 →"链接；下方「快捷入口」2 列宫格，每格 18px 圆角色块占位 + 12px 文字；不在业务页面重复出现，避免变成第二个导航
  • 监控/空数据类卡片：无数据时居中显示"暂无数据"文字（gray-5，不用插画），保持卡片高度不塌陷

表格与视觉基调

  • 表头浅灰小字（12px/650），行高宽松（≥44px），纯 1px 描边分隔，不用斑马纹、不用阴影
  • 数字类列右对齐、等宽数字；状态类列用色块徽章（见 04 节状态徽章）
  • 整体基调：纯白背景 + 极浅灰描边，强调色只用于当前态文字/图标/链接，色块面积始终很小

06 · 设计原则

  1. 信息优先：默认表格 + 描边卡片，不用大色块/大留白牺牲数据密度
  2. 色彩分工明确：teal 只做品牌强调，语义色（成功/警示/危险）独立，不与品牌色混用
  3. AI 是能力，不是频道：AI 输出内嵌在具体业务动作旁，永远可编辑/可重新生成，不单独占导航入口
  4. 身份切换是演示工具而非产品功能：视觉上弱化，不与真实权限模型混淆
  5. 加载态必须可见：拉取数据时显示骨架屏或占位条，禁止空白/"-"直接展示
