# 在线教育辅助教学系统 · 前端

面向高校教学场景的三端演示前端（学生 / 教师 / 管理员），参照《项目设计文档》与《UI 设计规范》，视觉采用**腾讯云控制台风格**（品牌蓝 `#1468e8`、纯白面板 + 1px 描边、表格优先、信息密集、低装饰）。

技术栈：Vue 3 + Vite + TypeScript + Vue Router + Pinia + frappe-ui + Tailwind + Lucide。默认使用本地演示数据，可独立运行，无后端耦合。

## 快速开始

```bash
npm install
npm run dev      # 本地开发（默认演示数据）
npm run build    # 生产构建
npm test         # 单元测试（vitest）
```

## 运行模式

- **演示模式（默认）**：全部数据来自 `src/mocks/data.ts`，交互（提交作业、发布成绩、审核课程、发布公告等）在浏览器内即时生效，刷新后会话角色保持。
- **真实模式**：设置环境变量 `VITE_API_MODE=real` 后才通过网关访问真实后端（`VITE_GATEWAY_URL`，默认 `http://localhost:18080`）；真实模式的错误会显式提示，不会静默回退到演示数据。

## 契约接口层（`src/services/api/`）

与 `api-reference.md` 的全部公开接口一一对应，按域拆分为 15 个模块（认证、文件、课程分类、教师课程、课程内容、学生学习、课程审核、用户授权、作业与成绩、论坛、预警、考试题库、公告、管理统计、AI），从 `@/services/api` 统一导出：

- 每个接口双模式：演示模式落到契约形状的本地数据库（`localStorage` 持久化、乐观锁 409、`resetDemoData()` 恢复种子），真实模式经网关请求并透传 `Authorization` / `X-Trace-Id`。
- `types.ts` 覆盖契约中全部 DTO/VO；分页统一 `PageResponse<T>`；错误统一抛出带 `code` 与 `traceId` 的 `RuntimeError`。
- AI 课程问答走 SSE（`meta/delta/citation/done/error`）；文档标注“契约待冻结”的三个 AI 接口（评语草稿、预警解读、组卷建议）在任何模式下都只用演示数据，不请求真实后端。

## 演示账号

登录页选择身份即可进入对应工作台（演示态，非真实鉴权）：

| 身份 | 落地页 | 主要能力 |
|---|---|---|
| 学生 王一诺 | `/student/dashboard` | 课程学习、章节 AI 答疑、作业提交、考试/成绩、课程互动 |
| 教师 李明 | `/teacher/dashboard` | 教学工作台、课程管理、作业批改（AI 评语草稿）、考试题库（AI 组卷建议）、学情预警、课程互动 |
| 管理员 周敏 | `/admin/dashboard` | 数据看板、用户管理、课程治理（驳回需填原因）、内容治理、数据统计 |

顶栏「预览身份」下拉可随时切换三端，导航、可访问路由与数据随之变化。

## 设计约束

- 品牌蓝只用于强调（当前导航、主按钮、链接、图表主色、焦点环）；语义色（成功/警示/危险）与品牌色区分。
- AI 是能力而非频道：答疑、评语草稿、风险解读、组卷建议均以内嵌卡片出现在对应业务动作旁，永远可编辑/重新生成，且不自动写入正式成绩、评语或试卷。
- 前端权限仅为体验优化，真实权限边界以后端为准。
