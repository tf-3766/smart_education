# 在线教育辅助教学系统文字版低保真原型

> 依据文档：[`design-research.md`](./design-research.md)、[`sitemap.md`](./sitemap.md)、[`ui-spec.md`](./ui-spec.md)  
> 表达方式：Markdown、ASCII 框图和文字说明  
> 当前阶段：只描述页面骨架、模块顺序、数据与交互，不包含 Vue 代码或图片。

## 0. 阅读约定

### 0.1 原型基线

- 线框以 1440px 左右桌面视口为主要参考，不追求像素级对齐。
- 普通应用页使用 64px 顶部栏、232px 左侧栏和 24–32px 内容区边距。
- 学生课程学习页可收起全局侧栏，使用“章节目录 + 学习内容 + AI 助手”三栏结构。
- 教师批改与智能组卷属于工作台页面，优先保证大屏连续操作效率。
- 管理员看板优先展示全局指标、异常和下钻入口，不堆装饰性图表。

### 0.2 ASCII 标记

```text
[主按钮]       当前页面最重要操作
[次按钮]       次要操作
[状态：...]    统一业务状态
[AI]           AI 生成、建议或解释区域
→              页面跳转或操作结果
```

### 0.3 AI 闭环统一规则

所有出现 AI 的页面都遵循以下流程：

```text
业务触发入口
  → 确认课程 / 章节 / 提交 / 预警 / 题库上下文
  → 正在检索资料或分析数据
  → 流式生成结果
  → 展示来源、依据或建议范围
  → 用户编辑 / 采纳 / 复制 / 重新生成
  → 人工确认保存、发布或进入下一业务动作
```

AI 超时、无结果或权限不足时，必须保留用户输入和已选上下文，并提供明确重试或返回操作。

## 1. 学生端首页 / 学习工作台

**一句话说明：** 帮助学生在进入系统后的第一屏看清“今天学什么、要交什么、哪里需要改进”。

- 所属角色：学生
- 建议路由：`/student/dashboard`
- 页面定位：低到中信息密度，任务优先，学习引导强于统计展示

### 1.1 页面目标

- 汇总个人课程、作业、考试、进度和成绩，而不是展示全校宏观数据。
- 让学生用一次点击进入继续学习、待交作业或考试说明。
- 对学习风险给出可理解的依据和可执行的下一步。
- 将 AI 学习建议放在真实进度和任务之后，避免成为无上下文聊天入口。

### 1.2 页面布局结构

```text
┌────────────────────────────────────────────────────────────────────────────────────────────┐

│                  │ [中风险] 大学英语近 3 次任务缺交 1 次，阅读章节进度低于课程计划          │
│                  │ [查看预警依据] [生成改进建议]                                             │
│                  │ [AI 建议结果区：本周完成 2 个具体学习动作 / 加入计划 / 向 AI 追问]        │
│                  ├──────────────────────────────────────────────┬──────────────────────────┤
│                  │ 作业与考试提醒                                │ 最新公告                 │
│                  │ 截止时间 / 开考时间 / 状态 / 快速入口         │ 课程 / 标题 / 发布时间   │
```

模块顺序：

1. 欢迎语与个人学习概览。
2. 继续学习 + 本周待办。
3. 学习进度 + 成绩趋势/学习时长。
4. 学习风险 + AI 学习建议。
5. 作业考试提醒 + 最新公告。

窄屏处理：顶部栏缩为 56px，左侧栏变为菜单抽屉；概览卡 2 列，其他区域全部上下排列；“继续学习”和“本周待办”保持前两位。

### 1.3 主要数据字段

| 模块 | 主要字段 |
|---|---|
| 个人概览 | 学生姓名、当前学期、在学课程数、本周待办数、即将考试数、本周学习时长 |
| 继续学习 | 课程 ID、课程名、教师、课程进度、已完成/总课时、当前章节、下一课时、最后学习时间 |
| 本周待办 | 任务类型、标题、课程、截止/开考时间、状态、优先级、目标路由 |
| 学习进度 | 课程名、完成百分比、计划进度、最近学习时间 |
| 成绩趋势 | 课程或总览范围、成绩值、发布时间、相较上次变化、未发布状态 |
| 学习时长 | 日期/周、有效学习时长、同比变化；无数据时不能显示虚假 0 趋势 |
| 风险提示 | 风险 ID、等级、课程、触发原因、证据摘要、生成时间、处理/已读状态 |
| AI 建议 | 预警/进度上下文、建议内容、依据、生成状态、是否加入计划 |
| 公告 | 公告 ID、标题、课程/校级范围、发布时间、未读状态 |

### 1.4 用户操作与交互流程

1. 学生进入首页，先加载个人概览骨架，再并行加载课程、任务、趋势和风险；某一模块失败不阻断其他模块。
2. 点击“继续学习”进入 `/student/courses/:courseId/lessons/:lessonId`；点击课程卡进入课程详情。
3. 点击待办作业进入作业详情；点击考试提醒进入考试说明，而不是直接开始考试。
4. 点击进度或成绩卡进入对应课程成绩/进度明细。
5. 点击“查看预警依据”进入 `/student/progress/warnings/:warningId`。
6. AI 学习建议闭环：
   - 业务入口：在风险卡或进度卡点击“生成改进建议”。
   - 生成过程：显示“正在分析本周进度、作业与成绩…”，允许取消。
   - 结果展示：给出 2–4 条具体行动及其数据依据，不给空泛鼓励语。
   - 后续操作：学生可“加入本周计划”“向 AI 追问”“重新生成”；加入计划需学生确认。
7. 若没有风险，风险区显示“当前未发现明显风险”，主操作改为“让 AI 帮我规划本周学习”，仍基于已选课程生成。

## 2. 学生端课程详情 / 章节学习页

**一句话说明：** 让学生在不离开课程上下文的情况下完成章节学习、查看资料与任务，并获得有来源的 AI 答疑。

- 所属角色：学生
- 课程详情路由：`/student/courses/:courseId`
- 章节学习路由：`/student/courses/:courseId/lessons/:lessonId`
- 页面定位：课程详情负责总览与入口，章节学习负责沉浸内容；本线框重点展示章节学习态

### 2.1 页面目标

- 始终让学生知道当前课程、章节位置、完成进度和下一步。
- 在一个学习空间内连接课程资料、作业、考试、摘要和笔记。
- AI 回答必须明确当前章节和资料范围，并可定位引用来源。
- 在较窄屏幕优先保证正文可读，不让 AI 侧栏挤压主要学习内容。

### 2.2 页面布局结构

```text

├────────────────────────────────────────────────────────────────────────────────────────────────┤

├──────────────────────┬───────────────────────────────────────────────┬─────────────────────────┤

│                      │ └───────────────────────────────────────────┘ │ AI 流式回答正文 [1][2]  │

│                      │ 相关资料与附件                               │                         │
│                      │ 课件.pdf / 示例代码 / 补充阅读 [预览/下载]   │ [有帮助] [复制] [重问]  │

```

课程详情概览态不展开 AI 对话，采用以下模块顺序：课程基本信息 → 继续学习 → 章节目录 → 近期作业/考试 → 公告与教师信息。点击具体章节后进入上图学习态。

窄屏处理：

- 992–1439px：AI 改为覆盖式抽屉。
- 768–991px：章节目录和 AI 都为抽屉，正文单列。
- <768px：AI 为全屏面板；资料、摘要和任务按正文下方顺序排列。

### 2.3 主要数据字段

| 模块 | 主要字段 |
|---|---|
| 课程基本信息 | 课程 ID、名称、教师、学期、分类、课程状态、已选课状态、总进度 |
| 章节目录 | 章节/课时 ID、标题、顺序、完成状态、解锁状态、是否含作业/测验 |
| 当前内容 | 课时标题、学习目标、内容类型、正文/视频地址、预计时长、完成条件、完成时间 |
| 课程资料 | 文件 ID、名称、类型、大小、版本、更新时间、预览/下载权限 |
| 课程任务 | 作业/考试 ID、标题、状态、截止/开考时间、目标路由 |
| 知识点摘要 | 摘要内容、发布教师、发布时间、来源课时、是否最新版本 |
| AI 上下文 | 课程 ID、课时 ID、选中文字、允许引用的资料范围 |
| AI 回答 | 会话 ID、问题、回答片段、生成状态、来源编号、反馈状态 |
| AI 来源 | 资料 ID、标题、资料类型、章节/页码、更新时间、定位锚点 |

### 2.4 用户操作与交互流程

1. 学生从课程详情选择章节，系统校验选课、章节发布和解锁状态后进入学习页。
2. 点击章节树切换课时；有未保存笔记时先自动保存或提示。
3. 学生查看正文/视频，满足完成条件后“标记完成”可用；完成后更新目录与总进度。
4. 点击资料进行预览/下载；无权限或已失效资料显示明确原因。
5. 点击作业/考试快捷入口进入带课程上下文的聚合页或详情页。
6. 点击“知识点摘要”查看教师已发布内容；未发布时不展示 AI 草稿。
7. AI 答疑闭环：
   - 业务入口：点击“向 AI 提问”、选中文字提问或快捷问题。
   - 上下文确认：面板顶部显示课程、课时和资料范围，学生可移除选中文字但不能越权扩展资料。
   - 生成过程：依次显示“确认章节范围 → 检索课程资料 → 生成回答”，支持停止。
   - 结果展示：回答正文内显示 `[1][2]`，底部列出可定位来源。
   - 后续操作：学生可复制、继续追问、重新生成、加入笔记或进入完整 AI 对话页。
8. 未找到依据时显示“未在当前课程资料中找到可靠依据”，由学生决定是否补充问题；系统不能假装已引用课程资料。

## 3. 教师端工作台

**一句话说明：** 帮助教师快速发现今天需要发布、批改和干预的事项，并进入对应课程工作流。

- 所属角色：教师
- 建议路由：`/teacher/dashboard`
- 页面定位：中高信息密度，工作队列优先，AI 入口必须先选择业务上下文

### 3.1 页面目标

- 汇总教师负责课程范围内的待处理事项和运行情况。
- 用数据卡片提供可行动摘要，而不是纯展示数字。
- 让教师快速进入课程管理、发布作业、批改、预警和组卷。
- AI 快捷入口只负责定位业务，不在首页生成脱离课程的内容。

### 3.2 页面布局结构

```text
┌────────────────────────────────────────────────────────────────────────────────────────────┐

├──────────────────┬─────────────────────────────────────────────────────────────────────────┤
```

### 3.3 主要数据字段

| 模块 | 主要字段 |
|---|---|
| 数据卡片 | 负责课程数、去重学生数、待批改提交数、待处理预警数、相应目标路由 |

### 3.4 用户操作与交互流程

8. AI 超时或失败时保留已有教师评分和评语，提示“AI 服务暂时不可用，批改数据未受影响”。

   - 人工处理：教师逐题采用、替换、排除或整体采用，再手工改顺序和分值。
   - 人工确认：通过总分、重复、超纲和覆盖校验后点击“确认试卷”。
4. 点击“重新生成”创建新版本，不覆盖教师已手工调整的当前版本；切换版本前提示保存。
5. 题库不足时不生成虚构题目，明确列出缺少的知识点/难度并引导补充题库或调整约束。

## 7. 学生学习风险预警页（简版）

**一句话说明：** 用可追溯证据解释学生当前学习风险，并把风险转化为学生能够确认和执行的改进计划。

- 所属角色：学生
- 建议路由：`/student/progress/warnings/:warningId`
- 页面定位：支持性、可行动，不用绝对化结论或只展示抽象风险分数

### 7.1 页面目标

- 告诉学生“为什么收到预警”，而不是只显示高/中/低标签。
- 连接相关课程、缺交作业、低分考试和落后章节。
- 由 AI 基于可见学习数据生成具体建议，并让学生确认加入计划。
- 保留预警历史和处理状态，不因“已读/已完成”而删除依据。

### 7.2 页面布局结构

```text
┌────────────────────────────────────────────────────────────────────────────────────────────┐
│ 面包屑：成绩与进度 / 学习预警 / 大学英语                     [返回进度总览]                │
├────────────────────────────────────────────────────────────────────────────────────────────┤
│ [中风险] 大学英语学习进度需要关注                                                      已读 │
│ 触发时间：7月6日 09:20  | 当前课程进度 55% | 计划进度 72% | 预警状态：待改进              │
├──────────────────────────────────────────────┬─────────────────────────────────────────────┤
│ 风险依据                                     │ AI 风险解释与改进建议                      │
│                                              │                                             │
│ 1. 近 3 次作业缺交 1 次                     │ [生成/更新 AI 改进建议]                    │
│    → 英语阅读作业 4 [查看作业]               │ 状态：正在分析进度、作业与成绩…            │
│ 2. 当前进度落后课程计划 17%                  │                                             │
│    → 第5章阅读 [继续学习]                    │ [AI 建议]                                  │
│ 3. 最近一次测验 68 分                        │ 本周建议完成：                             │
│    → 查看成绩与教师反馈                      │ □ 周二前补交阅读作业 4                     │
│                                              │ □ 完成第5章 2 个课时                       │
│ [近 4 周进度趋势] [作业完成趋势]             │ □ 周四完成一次 20 分钟复习练习             │
│                                              │                                             │
│ 数据范围：仅本人的课程、作业、成绩和进度     │ 依据：上述 3 项学习数据，不含他人比较       │
│                                              │ [加入本周计划] [向 AI 追问] [重新生成]      │
├──────────────────────────────────────────────┴─────────────────────────────────────────────┤
│ 我的处理记录                                                                                │
│ [标记已了解] [填写个人计划/备注] [完成后标记“已改善”]                                       │
│ 历史：7月6日 已查看 · 7月7日 已加入计划 · 等待后续进度更新                                  │
└────────────────────────────────────────────────────────────────────────────────────────────┘
```

窄屏处理：风险摘要 → 风险依据 → AI 建议 → 处理记录依次上下排列；趋势图保留文字摘要，作业和章节入口使用整行按钮。

### 7.3 主要数据字段

| 模块 | 主要字段 |
|---|---|
| 预警摘要 | warningId、课程、风险等级、状态、触发时间、更新时间、是否已读 |
| 风险依据 | 依据类型、相关对象 ID、数值、阈值/计划值、时间范围、可见说明、目标路由 |
| 趋势 | 周期、课程进度、计划进度、作业完成率、成绩；未发布成绩不计 0 |
| AI 上下文 | 本人课程进度、作业、已发布成绩、教师允许的课程规则 |
| AI 建议 | 建议版本、具体行动、目标日期、依据、生成状态、是否加入计划 |
| 学生处理 | 已读时间、个人备注、加入计划时间、完成状态、最后更新 |

### 7.4 用户操作与交互流程

1. 学生从首页风险卡或进度页进入详情，系统只读取本人可见学习数据。
2. 页面先展示风险等级和可追溯依据；点击依据进入具体作业、章节或已发布成绩。
3. AI 建议闭环：
   - 业务入口：点击“生成/更新 AI 改进建议”。
   - 上下文确认：列出将分析的本人进度、作业和成绩，不使用其他学生私人数据。
   - 生成过程：显示“正在分析进度、作业与成绩…”，支持停止。
   - 结果展示：给出 2–4 条有截止时间或完成条件的行动，并标明使用了哪些依据。
   - 后续操作：加入本周计划、向 AI 追问、重新生成；学生可编辑个人计划。
   - 人工确认：只有学生点击“加入本周计划”后，建议才转为个人任务。
4. 学生“标记已了解”只更新已读状态，不删除预警。
5. 完成计划后可标记“已改善”；系统根据后续数据更新风险状态，而不是由 AI 单独关闭预警。
6. AI 无可靠结果时保留风险依据，提示学生按原始任务入口行动或联系教师。

### 8.6 复用边界

- 复用同一组件不等于三端展示完全一致；例如 `MetricCard` 复用布局和状态，学生显示学习指标，管理员显示全局指标。
- 页面级组件负责业务编排，基础组件负责视觉一致；不要把整个学生首页封装成无法拆分的“万能 Dashboard”。
- AI 组件只复用生成状态、来源和结果操作，不复用越权上下文；每次仍由所在业务页面传入允许的数据范围。
- 组件必须同时覆盖加载、空、错误、无权限和禁用状态，避免每个页面临时补样式。
