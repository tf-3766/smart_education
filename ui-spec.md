# 在线教育辅助教学系统前端视觉规范

> 适用技术：Vue 3 + Vite + TypeScript + Element Plus  
> 依据文档：[`sitemap.md`](./sitemap.md)、[`wireframes.md`](./wireframes.md)、[`docs/three-person-development-plan.md`](./docs/three-person-development-plan.md)  
> 当前阶段：统一视觉、组件与交互规则；不包含具体 Vue 页面代码或依赖安装。

## 1. 规范目标

本规范用于保证学生端、教师端和管理员端在后续开发中使用同一套视觉语言、组件规则和状态语义。

系统整体采用“校园学习工作台”方向：现代、清晰、正式、有学习氛围，但不依赖夸张渐变、发光效果或无意义动画制造“科技感”。三端共用颜色、字体、间距、圆角、反馈和状态规则，仅调整信息密度与页面重心。

角色定位：

- 学生端先回答“我接下来要学什么、提交什么、哪里需要改进”。
- 教师端先回答“我接下来要发布、批改或干预什么”。
- 管理员端先回答“全局哪里有异常、哪些数据需要处理”。
- AI 先回答“它正在帮助哪一个教学任务”，而不是只展示一个孤立聊天框。

## 2. 总体设计原则

### 2.1 专业、简洁、易读

- 页面以内容和任务为主，装饰只用于建立层级，不抢占注意力。
- 每个页面只保留一个最突出的主操作，例如“提交作业”“发布作业”“保存并发布”。
- 标题、筛选、内容、操作保持稳定顺序；同类型页面使用同一页面骨架。
- 信息必须可扫描：状态、课程、截止时间、分数、风险原因等关键字段不能藏在弱提示文字中。
- 颜色、图标和文字共同表达状态，禁止只依赖颜色。

### 2.2 学生端更具学习感和引导性

- 使用较充足留白、课程封面、进度条、时间线和明确的“继续学习”操作。
- 首页优先级固定为：待办 > 继续学习 > 考试提醒 > 风险提醒 > 公告 > 次要统计。
- 课程学习页弱化全局管理感，突出章节树、正文、笔记和上下文 AI。
- 提示语采用支持性表达，例如“还有 2 项待完成”，避免制造焦虑的夸张文案。

### 2.3 教师端与管理员端偏效率和信息管理

- 教师端使用队列、表格、分栏工作台和批量筛选，强调连续处理效率。
- 管理员端使用筛选器、指标、趋势、异常队列和数据表，减少装饰图片。
- 教师与管理员页面允许更高信息密度，但不得通过缩小到 12px 以下或压缩行高来堆数据。
- 课程、作业、考试、学生和审核等对象的操作区位置保持一致。

### 2.4 AI 融入业务流程

- 智能答疑出现在章节学习页，并自动带入课程与章节上下文。
- 知识点摘要由教师在课时编辑页生成草稿、编辑并发布；学生只看已发布版本。
- AI 批改评语出现在真实批改工作台中，不建立孤立的“AI 批改中心”。
- 风险解释出现在学生进度页和教师预警详情中，并展示触发依据。
- 智能组卷建议出现在具体考试的试卷编排页，并使用教师有权访问的题库。
- 所有 AI 结果明确标记“AI 草稿”或“AI 建议”，人工确认后才成为正式业务数据。

### 2.5 应避免的设计

- 避免大面积紫蓝渐变、霓虹发光、玻璃拟态和持续动态背景。
- 避免每个模块都套一层卡片，形成“卡片里套卡片”的多重边框。
- 避免为不同菜单随机分配高饱和颜色。
- 避免一个页面同时出现三个以上主色按钮。
- 避免使用大幅装饰性插画挤压课程、作业和数据内容。
- 避免十几张同尺寸小图表、3D 饼图、无业务含义的同比数字。
- 避免只写“暂无数据”，不说明是无数据、无搜索结果、无权限还是加载失败。
- 避免为了显得智能而展示模型原始推理过程、长时间假加载或不可追溯的 AI 结论。
- 避免无意义的弹跳、翻转、视差和数字滚动动画。
- 避免逐页覆盖 Element Plus 样式；共性差异必须进入主题令牌或项目级组件。

## 3. 设计令牌

### 3.1 核心色彩

| 颜色角色 | 色值 | 适用场景 | 禁止用法 |
|---|---|---|---|
| 主色 | `#2557D6` | 主按钮、当前导航、关键链接、选中态、核心进度 | 不用于大面积页面背景 |
| 主色悬停 | `#1E46AD` | 主按钮、链接 hover | 不作为普通文字色 |
| 主色按下 | `#183A90` | 主按钮 active | 不长期保持在静态元素 |
| 主色浅底 | `#EEF3FF` | 选中行、主色轻标签、学生当前任务背景 | 不承载大段正文 |
| 辅助强调色 | `#0F8B7B` | 学习完成、正向进步、次要强调 | 不替代统一成功状态色 |
| 辅助强调浅底 | `#E8F7F3` | 完成提示、正向变化背景 | 不用于危险或警告状态 |
| AI 专用色 | `#6D5CE7` | AI 图标、AI 按钮、AI 草稿边框、AI 上下文标识 | 不用于普通课程、考试和成绩状态 |
| AI 悬停色 | `#5748C7` | AI 操作 hover | 不作为主品牌色 |
| AI 浅底 | `#F1EFFE` | AI 消息区、建议卡、草稿提示 | 不铺满整页 |
| AI 边框 | `#C9C1F5` | AI 草稿、来源和建议卡边界 | 不用于普通卡片边框 |

### 3.2 状态色

| 状态 | 主色 | 浅背景 | 浅边框 | 适用场景 |
|---|---|---|---|---|
| 成功 | `#16835B` | `#EAF7F1` | `#B9E3D0` | 已完成、已通过、已发布、保存成功、趋势上升 |
| 警告 | `#B86400` | `#FFF4E5` | `#F3D3A5` | 临近截止、待审核、待批改、中风险、需人工确认 |
| 危险 | `#C7374F` | `#FDECEF` | `#F2B8C2` | 逾期、失败、驳回、高风险、不可逆删除 |
| 信息 | `#2563EB` | `#EFF6FF` | `#BFDBFE` | 一般说明、已提交、待开始、系统提示 |
| 中性 | `#64748B` | `#F1F5F9` | `#E2E8F0` | 草稿、未开始、已关闭、未评估、禁用 |

状态标签默认使用“深色文字 + 浅背景 + 浅边框”，不使用整块高饱和实色。只有按钮和极少数关键徽标可以使用实色背景。

### 3.3 背景、文字与边框

| 颜色角色 | 色值 | 适用场景 |
|---|---|---|
| 页面背景 | `#F6F8FC` | 应用主背景、列表页外层 |
| 卡片/表面背景 | `#FFFFFF` | 卡片、表格、抽屉、弹窗、输入区 |
| 次级填充背景 | `#F8FAFC` | 表头、筛选区、只读区、轻分组背景 |
| 禁用背景 | `#F1F5F9` | 禁用输入、不可操作区域 |
| 主文字 | `#172033` | 页面标题、模块标题、关键数据 |
| 正文文字 | `#334155` | 正文、表格主要字段、表单值 |
| 次级文字 | `#64748B` | 时间、说明、课程教师、辅助字段 |
| 弱提示文字 | `#94A3B8` | 占位符、禁用说明、非关键元信息 |
| 默认边框 | `#E3E8F1` | 卡片、输入框、表格外边界 |
| 分割线 | `#EDF1F6` | 卡片内部、列表行、区块分隔 |
| 遮罩 | `rgba(15, 23, 42, 0.45)` | Dialog、Drawer 遮罩 |
| 焦点环 | `rgba(37, 87, 214, 0.28)` | 键盘焦点、输入 focus 外环 |

弱提示色不能承载截止时间、成绩、风险和错误原因。禁用态与只读态必须有文字说明，不能只降低透明度。

### 3.4 Element Plus 主题映射

后续实现时优先通过全局主题变量统一 Element Plus，而不是逐页修改组件内部样式。Element Plus 官方支持 SCSS 变量和 CSS 变量两种主题方式；项目选择其中一种并保持单一来源。

| 项目设计令牌 | Element Plus 变量建议 | 值 |
|---|---|---|
| 主色 | `--el-color-primary` | `#2557D6` |
| 成功 | `--el-color-success` | `#16835B` |
| 警告 | `--el-color-warning` | `#B86400` |
| 危险/错误 | `--el-color-danger`、`--el-color-error` | `#C7374F` |
| 信息 | `--el-color-info` | `#2563EB` |
| 页面背景 | `--el-bg-color-page` | `#F6F8FC` |
| 表面背景 | `--el-bg-color` | `#FFFFFF` |
| 主文字 | `--el-text-color-primary` | `#172033` |
| 常规文字 | `--el-text-color-regular` | `#334155` |
| 次级文字 | `--el-text-color-secondary` | `#64748B` |
| 占位/弱提示 | `--el-text-color-placeholder` | `#94A3B8` |
| 默认边框 | `--el-border-color` | `#E3E8F1` |
| 浅边框 | `--el-border-color-lighter` | `#EDF1F6` |
| 基础圆角 | `--el-border-radius-base` | `8px` |
| 小圆角 | `--el-border-radius-small` | `6px` |

AI 紫色不覆盖 Element Plus 的全局 `primary`。AI 组件使用项目级局部令牌，避免普通按钮、分页和选中态全部变成紫色。

### 3.5 间距

采用 4px 基础网格：

| 令牌 | 数值 | 常见用途 |
|---|---:|---|
| `space-1` | 4px | 图标内部微间距 |
| `space-2` | 8px | 图标与文字、紧密字段 |
| `space-3` | 12px | 同组控件、标签间距 |
| `space-4` | 16px | 表单行、筛选项、移动端区块 |
| `space-5` | 20px | 普通卡片内边距 |
| `space-6` | 24px | 区块间距、桌面卡片内边距 |
| `space-8` | 32px | 页面大区块、桌面页面内边距 |
| `space-10` | 40px | 重要章节分隔 |
| `space-12` | 48px | 登录页或空状态的大区块间距 |

同一语义组使用 8–12px，不同语义区块使用 24–32px。不得出现 13px、19px、27px 等无体系的随意间距。

### 3.6 圆角、阴影与层级

| 对象 | 圆角 | 阴影 |
|---|---:|---|
| 标签、状态徽标 | 6px | 无 |
| 按钮、输入、选择器 | 8px | 无 |
| 普通卡片、表格容器 | 12px | 默认无；使用 1px 边框 |
| 大卡片、AI 结果区 | 12–16px | 必要时使用轻阴影 |
| Dialog、Drawer | 16px（贴边侧可为 0） | `0 12px 32px rgba(23, 32, 51, 0.14)` |
| 可点击卡片 hover | 12px | `0 4px 16px rgba(23, 32, 51, 0.08)` |

- 卡片默认依靠背景与边框分层，不为所有卡片添加阴影。
- 禁止卡片内再放多个带阴影卡片。
- 浮层层级由 Element Plus 统一管理，业务页面不随意写极大层级值。

### 3.7 图标

- 首选 `@element-plus/icons-vue`；若其语义覆盖不足，可整体切换到一套统一开源线性图标库，但不能混用两套风格。
- 常用尺寸：侧栏 20px、按钮 16–18px、状态 16px、空状态 48–64px。
- 统一线性风格，线宽视觉接近 1.75–2px。
- 图标与文字间距 6–8px。
- 只有图标的按钮必须有可见悬停提示和无障碍名称。
- 不使用 Emoji 作为正式菜单、状态或按钮图标。
- 同一语义固定图标：完成用勾，警告用三角提示，错误用圆形关闭/感叹，时间用时钟，AI 用统一星光或魔法棒类图标。

### 3.8 动效

- hover 和颜色变化：120–160ms。
- 抽屉、弹窗、折叠：180–240ms。
- 进度更新可平滑过渡，但不做数字翻滚和弹跳。
- 页面切换不使用大幅滑动或 3D 翻转。
- 加载动画只表示真实等待；不得用固定时长假装“AI 正在深度思考”。
- 用户开启“减少动态效果”时，取消非必要动画。

## 4. 字体与排版规范

### 4.1 字体优先级

中文字体优先级：

`"PingFang SC", "Microsoft YaHei", "Noto Sans CJK SC", "Hiragino Sans GB", sans-serif`

英文字体与数字优先级：

`Inter, -apple-system, BlinkMacSystemFont, "Segoe UI", "Helvetica Neue", Arial, sans-serif`

建议完整字体栈以 Inter 和系统 UI 字体在前，中文字符自动回退到中文字体。项目不能依赖在线字体加载后才能保持基本排版。

成绩、时间、统计数字建议使用等宽数字特性，避免数值变化时左右跳动。

### 4.2 字号、行高与字重

| 层级 | 桌面字号 / 行高 | 字重 | 适用场景 |
|---|---|---:|---|
| 页面标题 | 24px / 32px | 600 | 页面主标题，如“我的课程” |
| 页面重要数字 | 28–32px / 36–40px | 600 | 平均分、完成率、待处理数量 |
| 模块标题 | 18px / 26px | 600 | “今日待办”“提交情况” |
| 卡片标题 | 16px / 24px | 600 | 课程名、作业名、图表标题 |
| 课程阅读正文 | 16px / 28px | 400 | 章节正文、长说明 |
| 默认正文 | 14px / 22px | 400 | 表单、普通页面文本 |
| 表格文字 | 14px / 20px | 400；关键列 500 | 表头、单元格 |
| 辅助文字 | 12px / 18px | 400 | 时间、来源、次要说明 |
| 按钮文字 | 14px / 20px | 500 | 默认按钮 |
| 标签文字 | 12px / 18px | 500 | 状态标签 |

约束：

- 正文不使用 500 以上字重铺满整段。
- 中文标题通常使用 600，不使用 700–900 的过黑效果。
- 重要状态、错误和截止时间不低于 14px。
- 表格辅助列允许 12px，但姓名、课程、成绩、状态保持 14px。
- 一段正文的最大可读宽度建议 680–760px；课程阅读区最大 800–880px。

### 4.3 段落与文本

- 普通段落间距 12px，长文段落间距 16px。
- 标题与首段间距 12px，模块标题与内容间距 16px。
- 列表项垂直间距 8px；教学步骤类列表使用 12px。
- 中文正文默认左对齐，不两端强制对齐。
- 超过两行的课程卡片标题截断并提供完整标题提示；正文不得无提示截断。
- 日期统一为 `YYYY-MM-DD HH:mm` 或自然语言“今天 23:59”，同一页面不混用多种格式。
- 分数统一显示“84 / 100”或“84 分”，不要同列混用百分比和原始分。

### 4.4 窄屏文字层级

| 层级 | 窄屏建议 |
|---|---|
| 页面标题 | 20px / 28px |
| 模块标题 | 16px / 24px |
| 卡片标题 | 15–16px / 22–24px |
| 正文 | 保持 14px / 22px |
| 课程正文 | 保持 16px / 26–28px |
| 辅助文字 | 最低 12px / 18px |

移动端通过减少边距和并列列数适配，不通过把所有字体缩小到难以阅读来适配。

## 5. 布局规范

### 5.1 应用框架

| 区域 | 桌面建议 | 窄屏建议 |
|---|---:|---:|
| 左侧导航展开宽度 | 232px；允许范围 224–240px | 变为覆盖式菜单抽屉 |
| 左侧导航收起宽度 | 64px | 不保留窄侧栏 |
| 顶部导航高度 | 64px | 56px |
| 页面左右内边距 | 32px | 平板 24px；手机 16px |
| 页面上下内边距 | 24–32px | 16–20px |
| 内容网格间距 | 24px | 平板 20px；手机 16px |
| 面包屑与标题间距 | 8–12px | 可隐藏非必要面包屑 |

侧栏默认使用白色背景和右侧分割线；当前菜单使用主色文字、浅主色背景和清晰图标。不要使用高饱和深色侧栏与彩色菜单混搭。

### 5.2 内容宽度

- 学生普通页面：内容容器最大 1440px，宽屏居中。
- 教师/管理员表格与工作台：允许流式铺开，建议最大 1680px；超过后居中并保留两侧空白。
- 课程正文：中央阅读列最大 800–880px。
- 章节目录：桌面 260–280px。
- AI 侧栏：桌面 400–440px。
- 教师批改：学生队列 220–240px，中央提交区弹性，评分区 340–380px。

### 5.3 页面标准分区

```text
页面面包屑（可选）
页面标题区：标题 + 说明 + 主操作
筛选/上下文区
页面内容区
批量操作/分页区
```

标准间距：

- 面包屑 → 标题：8px。
- 标题区内部：标题与说明 4–8px；标题与操作 16px。
- 标题区 → 筛选区：24px。
- 筛选区 → 内容区：16px。
- 内容区块之间：24px。
- 表格 → 分页：16px。
- 页面底部安全留白：32–48px。

标题区主操作位于右侧；窄屏改为标题在上、操作在下，主按钮可占满宽度。

### 5.4 筛选区

- 筛选容器使用白色或次级填充背景，内边距 16px，圆角 12px。
- 常用筛选最多显示 4 项：关键词、课程/学期、状态、时间范围。
- 高级条件折叠在“更多筛选”中，不能让筛选区占据首屏一半。
- 控件水平间距 12px，垂直换行间距 12px。
- “查询”和“重置”固定在筛选项末尾；自动查询与按钮查询只能选一种主要模式。

### 5.5 卡片网格

- 使用 12 栅格；桌面卡片间距 24px，窄桌面 20px，手机 16px。
- 课程卡片最小宽度 280px。
- ≥1200px：学生课程卡片通常 3–4 列；教师课程卡片 3 列。
- 768–1199px：2 列。
- <768px：1 列。
- 同一行卡片高度尽量一致；内容差异通过限制摘要行数而不是固定所有文字高度。
- 一个页面首屏卡片组不超过 2 个层级，更多内容使用列表或进入详情。

### 5.6 三端布局差异

| 维度 | 学生端 | 教师端 | 管理员端 |
|---|---|---|---|
| 页面密度 | 低到中 | 中到高 | 高 |
| 首页结构 | 待办时间线、继续学习、课程卡、公告 | 待处理队列、课程运行、批改进度 | 4 个以内 KPI、趋势、异常队列、明细入口 |
| 主要呈现 | 卡片、时间线、进度、日历摘要 | 表格、队列、分栏工作台 | 筛选、图表、表格、审计 |
| 装饰图片 | 允许少量原创课程封面 | 仅用于课程识别、文件预览 | 基本不用 |
| 课程学习 | 可收起全局侧栏，内容沉浸 | 内容编排树 + 编辑区 | 只读审核/运行视角 |
| AI 布局 | 章节右侧栏或全屏对话 | 批改/编辑/预警/组卷局部面板 | 用量、安全、索引和异常管理 |

## 6. Element Plus 使用策略

### 6.1 三层组件关系

| 层级 | 示例 | 使用原则 |
|---|---|---|
| Element Plus 原生组件 | Button、Form、Table、Dialog、Drawer、Tabs | 不为每页重复改基础样式，统一主题和尺寸 |
| 项目级基础组件 | `AppPageHeader`、`AppFilterBar`、`AppStatusTag`、`AppEmptyState` | 收口项目布局、状态、空态和反馈规则 |
| 业务组件 | `CourseCard`、`AssignmentStatusTag`、`AiChatPanel`、`RiskWarningCard` | 表达教育业务语义，不泄漏页面级临时样式 |

出现以下情况时才封装项目组件：

- 同一组合在 3 个及以上页面重复。
- 需要统一权限、状态、空态或交互规则。
- 具有明确业务语义，例如课程卡、批改面板、风险卡。

不要为了封装而创建只有一层透传的 `MyButton`、`MyInput`。

### 6.2 全局尺寸

- Element Plus 全局组件尺寸建议使用默认档，并通过主题统一到 36px 控件高度。
- 大号按钮/输入：40px，用于登录、提交、发布等关键操作。
- 默认按钮/输入：36px，用于大多数页面。
- 小号按钮：28–32px，用于表格行操作和紧凑工具区。
- 手机端主要点击目标不低于 44px；可通过外层点击区域满足，不必把所有视觉控件放大。

## 7. 常用组件规范

### 7.1 按钮

建议映射：Element Plus `Button`。

| 类型 | 场景 | 示例 |
|---|---|---|
| 主按钮 | 当前页面唯一主要动作 | 提交作业、发布作业、保存成绩 |
| 默认按钮 | 次要动作 | 预览、导出、取消筛选 |
| 文字/链接按钮 | 低层级行内动作 | 查看详情、展开更多 |
| 危险按钮 | 不可逆或高风险操作 | 删除课程、撤销发布 |
| AI 按钮 | 明确的 AI 生成动作 | 生成评语草稿、智能组卷建议 |

规则：

- 页面级主按钮最多 1 个；弹窗底部最多 1 个高强调确认按钮。
- 删除、驳回、清空等操作不能使用主蓝色。
- 表格行操作默认使用文字按钮或“更多”菜单，避免一行出现 4–5 个实心按钮。
- 图标在文字左侧，间距 6px；纯图标按钮必须有 Tooltip。
- 加载时保留原按钮宽度，禁用重复点击，并用“正在提交…”等具体文案。
- AI 按钮使用紫色图标、浅紫边框或紫色次按钮；除非页面主任务就是 AI 生成，否则不使用实心紫色大按钮。
- 禁用按钮旁应可找到不可用原因，不能让用户猜测。

### 7.2 表单

建议映射：Element Plus `Form`、`FormItem`、Input、Select、DatePicker、Radio、Checkbox、Switch。

- 业务编辑表单默认标签在控件上方；仅短筛选表单使用行内标签。
- 普通表单最大宽度 720px；单字段宽度通常 320–480px，长文本和上传区占满。
- 两列表单列间距 24px；宽度 <992px 时改为单列。
- 必填项使用统一星号和明确校验，不在标签中反复写“必填”。
- 校验消息紧贴字段下方，使用危险色和具体原因，例如“截止时间必须晚于发布时间”。
- 提交失败保留用户输入，并将焦点移到第一个错误字段。
- Switch 只用于立即生效的二元设置；草稿中的“是否发布”优先使用明确按钮或单选，避免误触。
- 只读值使用 Descriptions 或只读信息块，不把禁用 Input 当成详情展示。

### 7.3 表格

建议映射：Element Plus `Table`。

- 表头高度 44px，数据行建议 48–52px。
- 表头背景 `#F8FAFC`，表头文字 13–14px/600，数据文字 14px。
- 第一列通常为名称/学生/课程，最后一列为操作；必要时固定这两列。
- 状态列不只显示彩色点，必须显示完整状态文字。
- 重要数值右对齐，普通文字左对齐，操作列右对齐或居中。
- 超长内容使用两行截断或 Tooltip；不能在不知道完整值的情况下只显示省略号。
- 排序图标只出现在可排序列；默认排序必须可被用户理解。
- 批量选择后，在表格上方出现批量操作栏，并显示“已选择 N 项”。
- 默认不使用高对比斑马纹；超长数据表可使用极浅交替背景。
- 无数据、加载、错误必须占据表格内容区，不保留一张空壳表格。

### 7.4 分页

建议映射：Element Plus `Pagination`。

- 桌面列表默认显示：总数、每页数量、上一页、页码、下一页；数据量大时增加跳页。
- 表格常用每页数量：10、20、50、100。
- 课程卡片常用每页数量：12、24、48。
- 只有一页时可隐藏分页，但仍需显示总数的场景由业务决定。
- 分页位于表格/列表右下方，上方间距 16px。
- 手机端只保留上一页、简化页码、下一页；总数移到列表标题处。
- 切页后滚动到内容区顶部，并保留筛选条件。

### 7.5 标签与状态徽标

建议映射：Element Plus `Tag`、`Badge`。

- Tag 高度建议 24px，左右内边距 8px，字号 12px/500，圆角 6px。
- 业务状态使用 Tag；未读数量和通知数量使用 Badge。Badge 不能代替“高风险”“已逾期”等状态文字。
- 同一个对象同时显示的状态标签不超过 2 个；更多信息放到详情或 Tooltip。
- 状态颜色必须遵守第 12 章的状态映射。
- 可关闭筛选标签与不可交互状态标签在外观上应有区别。

### 7.6 卡片

建议映射：Element Plus `Card` 或项目级表面容器。

- 默认白底、1px 边框、12px 圆角、20–24px 内边距，不加常驻阴影。
- 可点击卡片 hover 时提升边框和轻阴影，并显示明确可点击光标。
- 卡片标题与操作在同一顶部行，内容区与标题间距 16px。
- 不在普通卡片中再嵌套完整带边框卡片；内部信息用分组、分割线或浅背景。
- 同组卡片保持标题位置、内边距和底部操作位置一致。
- KPI 卡片只显示一个主指标、一个比较信息和一个说明，不塞小图表、图标、趋势、排名等所有内容。

### 7.7 空状态

建议映射：Element Plus `Empty` 或 `AppEmptyState`。

空状态分为：

| 类型 | 推荐文案结构 | 操作 |
|---|---|---|
| 初始无数据 | “还没有发布作业” + 原因说明 | 有权限时提供“新建作业” |
| 搜索无结果 | “未找到符合条件的课程” + 当前筛选摘要 | “清除筛选” |
| 无权限 | “你没有查看此内容的权限” + 返回路径 | “返回工作台” |
| 已完成 | “本周任务已全部完成” + 正向说明 | “继续学习” |
| 加载失败 | “内容加载失败” + 简短原因 | “重新加载” |

- 空状态图形宽度 80–120px，使用低对比线性插图或图标。
- 不使用强装饰插画，也不在管理表格空状态中放大型人物插画。

### 7.8 加载状态

建议映射：Element Plus `Skeleton`、Loading、按钮 loading。

- 首屏卡片和详情使用接近最终结构的 Skeleton，避免布局跳动。
- 表格刷新使用表格区域加载遮罩或 5–8 行骨架，不遮住整个应用。
- 按钮请求使用按钮内 loading，并保持按钮宽度。
- 短于约 200ms 的请求不必立即闪烁全屏加载；持续等待再展示加载状态。
- 加载超过正常时间时显示阶段或取消/重试入口，尤其是上传和 AI 生成。
- 禁止只有一个长期旋转图标而没有说明。

### 7.9 错误与反馈

| 反馈层级 | 推荐组件 | 场景 |
|---|---|---|
| 短暂操作反馈 | Message | 保存成功、复制成功、简单失败 |
| 页面内持续问题 | Alert | 数据部分失败、考试规则、AI 无依据 |
| 后台/跨页面通知 | Notification | 导出完成、批量任务完成 |
| 页面级错误 | Result / `AppErrorState` | 403、404、整页数据无法加载 |
| 字段错误 | FormItem 校验 | 表单输入不合法 |

- 错误文案包含“发生了什么 + 是否保留数据 + 下一步怎么做”。
- 不显示原始堆栈、接口地址或数据库错误。
- 保存失败时不能清空表单。
- Message 适合操作反馈，Notification 适合被动系统通知；不要同时弹出两者表示同一件事。

### 7.10 确认弹窗

建议映射：Element Plus `MessageBox` 或 `Dialog`。

- 仅对不可逆、高影响或易误触操作确认；普通查看、切换标签不弹窗。
- 宽度建议 420–480px；复杂影响说明使用 520–640px Dialog。
- 标题写清动作，例如“确认发布成绩？”而不是“提示”。
- 正文列出影响范围，例如“将向 46 名学生公开本次成绩”。
- 取消按钮在左，确认按钮在右；危险操作确认按钮使用危险色，并默认聚焦取消。
- 有未保存内容时，关闭 Drawer/Dialog 前提示“放弃修改”或“继续编辑”。
- 输入名称二次确认只用于删除课程、批量删除用户等极高风险操作。

### 7.11 抽屉

建议映射：Element Plus `Drawer`。

| 用途 | 桌面宽度 |
|---|---:|
| 快速详情/筛选 | 400–480px |
| AI 聊天侧栏 | 400–440px |
| 编辑表单 | 560–640px |
| 复杂上下文 | 不使用 Drawer，进入独立页面 |

- Drawer 包含固定头部、可滚动正文和固定底部操作。
- 不嵌套 Drawer；AI 来源在侧栏内展开或进入完整 AI 页。
- Drawer 关闭后再次打开应按业务决定保留草稿，不能无提示丢失输入。
- 手机端 Drawer 改为全屏，顶部提供明确返回/关闭按钮。

### 7.12 Tabs

建议映射：Element Plus `Tabs`。

- 用于同一对象下的相关信息，例如课程概览、章节、学生、成绩。
- 一级 Tabs 建议 2–6 个；超过 6 个应重新分组或使用侧栏。
- 默认使用简洁线性 Tabs，不使用多层 `border-card`。
- 关键业务 Tabs 与路由/查询参数同步，刷新与返回后保持当前项。
- 标签中可显示数量，如“待批改 18”，但不显示多个彩色徽标。
- Tabs 下方内容与导航间距 16–24px。

### 7.13 搜索与筛选栏

- 搜索框宽度 240–320px，placeholder 具体，例如“搜索课程名称/教师”。
- 关键词输入后按 Enter 或点击查询；不要部分页面自动查、部分页面按钮查而无规律。
- 活跃筛选条件以可关闭标签显示，并提供“清除全部”。
- 筛选条件进入 URL 查询参数，返回列表时恢复。
- 手机端默认显示搜索和一个“筛选”按钮，其他条件放入全屏/底部筛选面板。

### 7.14 图表区

- 单图高度通常 280–320px；小趋势图 120–160px。
- 优先折线、水平条形、堆叠条形；饼图最多 4–5 类，禁止 3D 图。
- 图表标题下方标注时间范围、单位和筛选范围。
- 图例顺序与数据顺序一致；同一指标跨页面保持同色。
- Tooltip 显示完整日期、数值、单位和对比，不只显示数字。
- 无数据时显示原因，不绘制一条 0 线冒充数据。
- 图表必须提供表格明细或可下钻入口，不能成为唯一数据载体。
- 管理员首页首屏最多 4 个 KPI 和 2–3 张主要图表。

### 7.15 文件上传区

建议映射：Element Plus `Upload`。

- 上传前显示允许格式、单文件大小、数量和用途。
- 支持点击与拖拽时，拖拽区高度建议 120–160px。
- 文件列表显示文件名、大小、上传进度、结果、预览/删除/重试。
- 上传失败不删除其他成功文件；失败项保留并允许重试。
- 作业“文件上传完成”与“正式提交作业”是两个状态，必须有独立确认。
- 删除已上传文件需依据是否已提交决定是否确认。
- 不在前端文案中承诺后端未支持的格式或大小。

## 8. 教育业务组件规范

### 8.1 课程卡片

建议组件：`CourseCard`。

结构顺序：

1. 课程封面或低饱和分类色区域。
2. 课程名称，最多两行。
3. 教师、学期/课程分类。
4. 学习进度或课程状态。
5. 下一项任务/继续学习操作。

规范：

- 卡片宽度不低于 280px；封面建议 16:9，桌面高度约 132–152px。
- 同一课程使用稳定封面/分类色，不能每次随机变化。
- 学生卡片优先显示进度和下一任务；教师卡片优先显示学生数、待批改和课程状态；管理员默认使用表格而不是课程卡片墙。
- “继续学习”是学生在学课程卡的主操作；未选课程显示“查看课程/选课”。
- 进度条必须配合“已完成 8 / 20 课时”文字。

### 8.2 作业状态卡片

建议组件：`AssignmentStatusCard`、`AssignmentStatusTag`。

结构：作业名、课程、截止时间、状态、分数/反馈、主操作。

- 待完成卡片突出截止时间和“去完成”。
- 已提交卡片显示提交时间和“等待批改”，不显示空分数。
- 已批改卡片显示分数、评语摘要和“查看反馈”。
- 已逾期使用危险色标签和具体逾期时长，但整张卡片不铺红色。
- 卡片列表在教师/管理员端优先改为表格或队列，避免高密度场景卡片化。

### 8.3 成绩趋势卡片

建议组件：`ScoreTrendCard`、`ScoreTrendChart`。

- 主数字显示当前成绩或平均分，28–32px/600。
- 趋势显示“较上次 +3.5”或“近 4 次稳定”，并配箭头和文字。
- 上升使用成功色，下降使用危险色，稳定使用中性色；不能只靠红绿箭头。
- 趋势图最多 6–12 个点；完整成绩仍进入明细表。
- 未发布成绩显示“未发布”，不能按 0 分绘图。
- 排名默认不突出；确需显示时使用“前 30%”等区间并说明范围。

## 9. AI 功能组件规范

### 9.1 AI 视觉身份

- AI 专用色为 `#6D5CE7`，只用于 AI 图标、边框、按钮和来源提示。
- AI 组件顶部必须出现“AI 助手”“AI 草稿”或“AI 建议”文字，不只使用星光图标。
- AI 区域使用浅紫底/边框建立轻微区别，不能满屏渐变、发光或机器人吉祥物化。
- AI 内容与正式业务内容在视觉上可区分；教师确认发布后改用普通业务样式，并保留生成来源记录。

### 9.2 AI 答疑聊天窗口

建议组件：`AiChatPanel`。

桌面侧栏结构：

1. 头部 56px：标题、关闭/完整页入口。
2. 上下文条：课程、章节、资料范围，可展开查看。
3. 快捷问题：解释概念、给一个提示、生成练习、总结本节。
4. 消息列表：用户问题、AI 回答、来源、反馈操作。
5. 固定输入区：输入、附件/引用、发送、停止生成。

视觉规则：

- 用户消息使用主色浅底气泡，最大宽度 80%。
- AI 回答使用白底或无气泡正文，左侧显示统一 AI 标识。
- 消息间距 16px，同一回答段落间距 8–12px。
- 输入区最小高度 88–96px，最大高度不超过面板的 40%；超出后内部滚动。
- 输入框上方显示当前上下文，例如“数据结构 > 第 3 章 > 二叉树”。
- 未选择课程上下文时，先提示选择，不允许悄悄使用全局未知资料回答。

### 9.3 流式回答与生成状态

状态顺序建议：

| 状态 | 界面文案 | 表现 |
|---|---|---|
| 校验上下文 | “正在确认课程与章节范围…” | 轻量状态行 |
| 检索资料 | “正在检索本章课程资料…” | 来源图标 + 轻动画 |
| 生成中 | “正在生成回答…” | 流式文字 + 停止按钮 |
| 已完成 | 不再显示加载文案 | 出现来源和结果操作 |
| 已停止 | “已停止生成” | 保留已生成内容，可继续/重新生成 |
| 失败 | 明确错误原因 | 保留问题与上下文，提供重试 |

- “思考中”只表示等待状态，不展示模型内部推理链。
- 流式文本末尾可使用低调光标，不使用全屏粒子或长时间三点假动画。
- 用户可停止生成；停止后保留已生成部分。
- 页面切换前若仍在生成，提示是后台继续还是停止；首版可统一停止并保留会话记录。

### 9.4 课程资料引用

建议组件：`AiSourceList`、`AiSourceItem`。

- 回答中使用 `[1]`、`[2]` 等短编号，回答末尾统一列出来源。
- 每条来源显示：资料类型、标题、章节/页码或课时、更新时间。
- 点击来源在当前面板内展开摘要或定位课程内容；不要在 AI Drawer 中再打开嵌套 Drawer。
- 明确标记“依据课程资料”“使用通用知识补充”或“未找到可靠依据”。
- 学生无权访问的资料不能出现在来源标题、摘要或链接中。
- 引用失效时显示“该资料已更新或不可访问”，不能打开空白页。

### 9.5 一键生成评语按钮

建议组件：`AiGenerateCommentButton`、`AiDraftEditor`。

- 位于教师批改评分面板的评语区附近，按钮文字为“生成 AI 评语草稿”。
- 使用紫色图标 + 浅紫边框的次按钮，不抢过“发布评分”。
- 生成前至少需要：学生提交、评分量规/问题点或教师选择的评价依据。
- 生成中按钮保持宽度，并显示“正在生成评语…”。
- 结果进入可编辑草稿区，顶部标记“AI 草稿”；教师修改后再点击“保存草稿”或“发布评分”。
- AI 不自动填写分数，不自动发布，不覆盖教师已有评语。

### 9.6 学习预警风险卡片

建议组件：`RiskWarningCard`、`RiskLevelTag`、`RiskEvidenceList`。

结构：风险等级、触发原因、证据、影响范围、建议动作、处理状态。

- 使用左侧 4px 风险色边或顶部标签，不整卡铺红/橙色。
- 高风险必须写明依据，例如“近 3 次作业缺交 2 次”，不能只显示“风险值 0.87”。
- 学生侧文案强调下一步行动；教师侧增加处理记录、联系/干预建议。
- AI 解释标记为建议；教师可采纳、修改或忽略。
- 风险关闭表示已处理，不删除历史证据。

### 9.7 智能组卷建议卡片

建议组件：`AiPaperSuggestionCard`、`PaperDistributionChart`。

显示内容：

- 建议依据：课程、知识点范围、题库范围。
- 题型分布：选择、填空、简答等。
- 难度分布：易/中/难比例。
- 总题数、总分、预计时长。
- 重复题、超纲、题库不足等警告。

交互：

- 教师可逐项“采用、替换、排除”，也可整体应用后继续手工编辑。
- 应用建议前显示将新增/替换的题目数量。
- 被教师手动修改的题目不能在重新生成时静默覆盖。
- 题库不足时说明缺少的知识点或难度，不生成不可追溯的虚构题目。

### 9.8 AI 结果操作

建议组件：`AiResultActions`。

| 操作 | 规则 |
|---|---|
| 编辑 | 教师侧摘要、评语、干预建议、试卷建议可编辑；学生不能编辑 AI 回答本身，但可编辑问题后重问 |
| 复制 | 复制后用短消息提示“已复制”；保留纯文本结构，不复制隐藏标记 |
| 重新生成 | 生成新版本，不静默覆盖已编辑内容；存在未保存修改时先确认 |
| 保存草稿 | 只保存到草稿状态，显示“已保存 HH:mm” |
| 应用/发布 | 与保存分离；明确影响范围并由有权限用户确认 |
| 反馈 | 提供“有帮助/需改进”及可选原因，不打断主流程 |

### 9.9 AI 异常与无结果

| 情况 | 推荐提示 | 操作 |
|---|---|---|
| 超时 | “本次生成超过等待时间，已保留你的输入。” | 重新生成、稍后再试 |
| 服务不可用 | “AI 服务暂时不可用，课程与作业数据未受影响。” | 重试、关闭面板 |
| 无可靠资料 | “未在当前课程资料中找到可靠依据。” | 补充问题、扩大允许的资料范围 |
| 权限不足 | “当前资料不在你的访问范围内。” | 返回当前课程或联系教师 |
| 内容受限 | “考试进行期间不能直接提供答案，可查看相关概念提示。” | 查看提示、返回考试 |
| 部分生成失败 | “回答未完整生成，已保留现有内容。” | 继续生成、重新生成 |

错误提示保留用户输入、已选上下文和已生成部分；不得把 AI 错误包装成普通“暂无数据”。

## 10. 数据与状态展示规范

所有状态必须由后端状态映射到统一中文文案；页面不得自行创造同义词，例如同一状态同时叫“已交”“提交完成”“已提交”。

### 10.1 作业状态

| 标准文字 | 颜色 | 图标语义 | 使用说明 |
|---|---|---|---|
| 草稿 | 中性 | 文档/编辑 | 仅教师可见，尚未发布 |
| 待完成 | 主色 | 编辑/时钟 | 学生可开始完成，未提交 |
| 已保存草稿 | 信息 | 保存/文档 | 已有未正式提交内容 |
| 已提交 | 信息 | 上传完成/勾 | 学生已正式提交，等待后续状态 |
| 待批改 | 警告 | 时钟 | 教师尚未完成批改 |
| 已批改 | 成功 | 勾 | 分数或反馈已完成；是否对学生可见另由发布状态决定 |
| 已退回 | 警告或危险 | 返回/警示 | 允许修改重交时用警告；明确失败时用危险 |
| 已逾期 | 危险 | 时钟警示 | 超过截止时间且未有效提交 |
| 已关闭 | 中性 | 锁/关闭 | 不再允许提交或修改 |

截止前 24 小时可加警告提示“即将截止”，但作业主状态仍是“待完成”。

### 10.2 考试状态

| 标准文字 | 颜色 | 图标语义 | 使用说明 |
|---|---|---|---|
| 草稿 | 中性 | 文档 | 教师未发布 |
| 待开始 | 信息 | 日历/时钟 | 已发布但未到开考时间 |
| 进行中 | 主色 | 播放/计时 | 当前可进入或正在答题 |
| 已交卷 | 成功 | 勾 | 学生完成交卷，结果可能尚未发布 |
| 待阅卷 | 警告 | 编辑/时钟 | 存在待处理主观题 |
| 成绩已发布 | 成功 | 成绩/勾 | 学生可查看结果 |
| 已结束 | 中性 | 关闭/日历 | 时间结束且业务关闭 |
| 已取消 | 危险 | 关闭 | 考试被取消，并显示原因 |

倒计时是考试进行中的辅助信息，不替代“进行中”状态。剩余 5 分钟等阈值提醒需要文字、声音/视觉策略由考试规则统一决定。

### 10.3 课程状态

| 标准文字 | 颜色 | 使用说明 |
|---|---|---|
| 草稿 | 中性 | 教师编辑中 |
| 待审核 | 警告 | 已提交管理员审核 |
| 审核驳回 | 危险 | 必须同时显示驳回原因 |
| 选课开放 | 信息 | 学生可选课 |
| 进行中 | 主色 | 课程正在教学运行 |
| 已完成 | 成功 | 学生完成课程或课程教学完成，按上下文区分文案 |
| 已结束 | 中性 | 课程周期结束，只读访问 |
| 已下线 | 危险/中性 | 管理动作导致不可见；面向学生可只显示“不可用” |

### 10.4 学习风险等级

| 等级 | 颜色 | 卡片表现 | 文案原则 |
|---|---|---|---|
| 未评估 | 中性 | 普通边框 | “暂无足够数据评估” |
| 低风险 | 成功 | 成功色标签 | 强调保持当前节奏 |
| 中风险 | 警告 | 左侧警告边 + 证据 | 强调可执行改进项 |
| 高风险 | 危险 | 左侧危险边 + 证据 | 明确原因、影响和人工干预入口 |

不展示“你一定会挂科”等绝对化结论；风险颜色不能脱离证据单独出现。

### 10.5 成绩等级与趋势

- 具体等级阈值由学校/课程规则配置，前端不写死 90/80/60。
- 分数本身使用主文字或主色；只有不及格/异常才使用危险色。
- “优秀/良好/及格/不及格”必须同时显示文字，不使用纯颜色等级。
- 趋势上升：成功色 + 向上箭头 + “较上次 +3.5”。
- 趋势下降：危险色 + 向下箭头 + “较上次 -2.0”。
- 趋势稳定：中性色 + 水平箭头 + “近 3 次稳定”。
- 未发布、缺考、缓考、免考为独立业务状态，不能当作 0 分。

### 10.6 审核状态

| 标准文字 | 颜色 | 图标语义 | 使用说明 |
|---|---|---|---|
| 草稿 | 中性 | 编辑 | 尚未提交审核 |
| 待审核 | 警告 | 时钟 | 进入管理员队列 |
| 审核中 | 信息 | 查看/进行 | 有审核人正在处理时使用 |
| 已通过 | 成功 | 勾 | 允许进入后续发布流程 |
| 已驳回 | 危险 | 关闭/警示 | 同时显示审核意见 |
| 已撤回 | 中性 | 返回 | 提交者主动撤回 |

审核标签与课程运行状态可以同时存在，但最多显示两个标签，例如“已通过 / 选课开放”。

## 11. 响应式规范

### 11.1 断点

为便于使用 Element Plus 的响应式栅格，采用其 xs/sm/md/lg/xl 分段思路：

| 断点 | 宽度 | 本项目策略 |
|---|---:|---|
| xs | <768px | 手机单列、全屏抽屉、简化表格 |
| sm | ≥768px | 小平板，两列卡片、覆盖式侧栏 |
| md | ≥992px | 平板/窄桌面，可显示部分双栏 |
| lg | ≥1200px | 桌面标准布局、固定侧栏 |
| xl | ≥1920px | 保持最大内容宽度，不无限拉伸 |

桌面端（≥1200px）是教师端和管理员端首要目标；学生端必须保证手机上的学习、提交和成绩查看可用。

### 11.2 导航适配

- ≥1200px：固定 232px 侧栏，可收起到 64px。
- 768–1199px：默认收起或使用覆盖式侧栏；内容区不为侧栏永久预留过多空间。
- <768px：顶部 56px，侧栏变为全屏/覆盖菜单。
- 学生手机端可使用最多 4 项底部导航：首页、课程、任务、我的；AI 仍从课程/章节进入，不占用固定底栏主位。
- 教师和管理员手机端使用菜单抽屉，不复制整套桌面侧栏。

### 11.3 页面与卡片适配

- 页面标题的标题、说明、操作在手机端上下排列。
- KPI 卡：桌面 4 列，平板 2 列，手机 1–2 列；主数字不得缩到难读。
- 筛选栏：桌面行内，手机只显示搜索和“筛选”按钮，其余进入筛选面板。
- 课程详情的主内容和近期事项从左右两列改为上下排列。
- 章节学习：先收起 AI，再将章节目录变为抽屉；正文保持 16px 阅读字号。
- 教师批改三栏在窄屏拆为“学生列表 → 提交内容 → 评分”的分步视图，首版不要求手机端高效连续批改。

### 11.4 表格适配

以下表格优先保留表格并允许横向滚动：

- 教师作业提交情况、课程成绩册、题库、考试记录。
- 管理员用户、课程、审核、统计明细、审计记录。
- 需要比较多列的学生课程成绩构成。

横向滚动规则：

- 固定首列（名称/学生）和末列（操作）。
- 表格上方提示“左右滑动查看更多”。
- 不把 10 列数据强行压缩进手机宽度。
- 学生作业、考试、公告等简单列表在手机端改为卡片/列表行，不使用横向表格。

### 11.5 AI 小屏展示

- ≥1440px：AI 可作为 400–440px 固定右侧栏，与课程内容同屏。
- 992–1439px：AI 使用 420–480px 覆盖式 Drawer，不压缩正文到不可读。
- 768–991px：Drawer 宽度约 70–80vw，上下文区域可折叠。
- <768px：AI 变为全屏页面/全屏 Drawer，顶部显示返回、标题和上下文。
- 小屏输入区固定底部并考虑安全区；消息区独立滚动。
- 来源列表默认折叠，点击后在当前页面展开，不再叠加第二层抽屉。

## 12. 可访问性与交互底线

- 正文与背景保持清晰对比；浅色文字只承载非关键信息。
- 所有可操作元素支持键盘焦点，焦点环使用主色半透明外环，不能直接移除。
- 图标按钮、图表、上传区和状态点提供文字名称。
- 表单错误与对应字段关联；错误出现后焦点可定位。
- Dialog/Drawer 打开时焦点进入浮层，关闭后返回触发元素。
- 流式 AI 回答使用适度的实时播报策略，不能让屏幕阅读器逐字打断。
- 图表信息必须能通过表格或文字摘要获取。
- 成功/警告/危险均使用“颜色 + 图标 + 文字”。
- 桌面点击目标建议不小于 36×36px，手机主要目标不小于 44×44px。
- 考试倒计时、截止时间和风险提示不能只通过动画或颜色提醒。

## 14. 标准页面组合

### 14.1 列表管理页

`AppPageHeader → AppFilterBar → 批量操作栏（按需）→ Table/卡片列表 → Pagination`

适用：教师作业、考试、题库；管理员用户、课程、审核、公告。

### 14.2 学生任务页

`AppPageHeader → 状态标签筛选 → 近期/逾期任务 → 任务卡片列表 → 空态/分页`

适用：学生作业、考试、公告。

### 14.3 详情页

`面包屑 → 标题与状态 → 关键摘要 → Tabs/分区正文 → 底部或固定操作`

适用：课程详情、作业详情、考试说明、用户详情、课程审核详情。

### 14.4 教师工作台页

`上下文头部 → 左侧队列 → 中央工作内容 → 右侧处理面板`

适用：作业批改、试卷编排。窄屏拆为分步视图。

### 14.5 AI 增强页

`原业务页面 → AI 入口 → 上下文确认 → 生成/流式状态 → 来源 → 编辑/采纳/保存`

AI 区域不能覆盖原业务的保存、提交、发布和权限校验。

## 15. 前端视觉检查清单

### 15.1 设计令牌

- [ ] 是否只使用规范中的主色、状态色、文字色和边框色？
- [ ] 是否使用 4px 间距体系，没有随意出现 13px、19px 等值？
- [ ] 控件、卡片、浮层圆角是否符合 8px / 12px / 16px 层级？
- [ ] 是否避免逐页覆盖 Element Plus 内部样式？
- [ ] 是否存在可以收口为主题变量或项目级组件的重复样式？

### 15.2 页面结构

- [ ] 页面是否按标题区、筛选区、内容区、操作/分页区组织？
- [ ] 是否只有一个最突出的页面主操作？
- [ ] 学生端是否突出下一步学习行动，而不是管理统计？
- [ ] 教师端是否支持连续处理和课程范围筛选？
- [ ] 管理员端是否突出全局异常和下钻入口，而非课程学习体验？
- [ ] 详情页刷新后是否仍有完整标题、状态、操作和上下文？

### 15.3 组件一致性

- [ ] 同类按钮的类型、尺寸、图标位置是否一致？
- [ ] 表单标签、必填、校验、保存和取消位置是否一致？
- [ ] 表格表头、行高、状态列和操作列是否一致？
- [ ] 状态是否使用统一中文文案、颜色和图标？
- [ ] 卡片是否避免多层嵌套和过度阴影？
- [ ] Tabs 是否不超过 6 个，并与路由/状态同步？

### 15.4 完整状态

- [ ] 是否有首次加载和局部刷新状态？
- [ ] 是否有初始空数据和筛选无结果两种空状态？
- [ ] 是否有接口失败、重试和数据保留说明？
- [ ] 是否有登录失效、权限不足、资源不存在和资源状态不允许的不同提示？
- [ ] 禁用按钮是否说明原因？
- [ ] 未发布成绩、缺考和暂无数据是否没有错误显示为 0？

### 15.5 AI 业务价值

- [ ] AI 是否从课程、章节、提交、预警或试卷等明确业务入口进入？
- [ ] 是否显示当前课程、章节、资料或题库上下文？
- [ ] AI 回答是否展示来源或明确“未找到依据”？
- [ ] 是否提供生成中、停止、超时、失败和无结果状态？
- [ ] AI 摘要、评语、预警建议和组卷建议是否标记为草稿/建议？
- [ ] 是否允许编辑、复制、重新生成和保存，并避免覆盖未保存修改？
- [ ] 是否必须人工确认后才发布评语、摘要、成绩或试卷？
- [ ] AI 权限是否继承原业务页面，没有扩大数据范围？

### 15.6 响应式与可访问性

- [ ] 1200px、992px、768px 以下是否有明确布局变化？
- [ ] 手机端是否通过堆叠和抽屉适配，而不是只缩小字体？
- [ ] 高密度表格是否可横向滚动并固定关键列？
- [ ] AI 侧栏在小屏是否转为全屏并保留上下文？
- [ ] 所有状态是否同时使用文字、图标和颜色？
- [ ] 图标按钮是否有 Tooltip 和无障碍名称？
- [ ] 键盘焦点是否清晰，Dialog/Drawer 焦点是否正确返回？
- [ ] 图表是否有文字摘要、明细或下钻入口？
